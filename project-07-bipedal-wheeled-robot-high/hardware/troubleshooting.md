# 硬件故障排除 | Hardware Troubleshooting

本文档按故障类别分类，每种问题提供：症状 → 可能原因 → 解决方案。

> This document is organized by fault category. Each issue provides: Symptom -> Possible Cause -> Solution.

---

## 一、电源问题 | Power Issues

### 1.1 机器人无法开机 / Robot Won't Turn On

**症状 Symptom**：打开电源开关后，ESP32-S3 指示灯不亮，电机无反应。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 电池没电 | 用万用表测量电池电压，3S 锂电不应低于 10.2V（单节 3.4V）。低于 9.9V 需充电 |
| 2 | 电源开关故障 | 用万用表测量开关两端通断，确认开关完好 |
| 3 | XT60 插头接触不良 | 检查插头是否插紧，金属触点是否氧化。用酒精棉清洁 |
| 4 | 保险丝/保护电路触发 | 检查驱动板上是否有保险丝，是否熔断 |
| 5 | 电源线焊接点脱落 | 目视检查所有焊接点，重新焊接松脱的接头 |
| 6 | 电池过放保护（BMS 板） | 部分电池自带保护板，过放后需充电恢复 |

### 1.2 电池续航时间短 / Battery Drains Fast

**症状 Symptom**：电池满电但只能运行 10 分钟以内。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 电池容量不足 | 1500mAh 电池正常运行约 15-30 分钟，属正常范围 |
| 2 | 电池老化/内阻增大 | 更换新电池。锂电池寿命一般 300-500 次循环 |
| 3 | 舵机堵转消耗大电流 | 检查舵机是否受阻，一个堵转舵机可消耗 2A+ |
| 4 | 接线短路 | 检查是否有裸线接触，用万用表测量静态电流 |
| 5 | 电池标称容量虚标 | 购买品牌电池（格式/花牌/达尔普等），避免虚标 |

### 1.3 电压骤降导致重启 / Voltage Drop Causes Reset

**症状 Symptom**：机器人运动时 ESP32-S3 突然重启。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 电池内阻大，大电流时压降严重 | 使用 25C 以上放电倍率的电池 |
| 2 | 电源线过细 | 使用 18AWG 或更粗的硅胶线 |
| 3 | ESP32-S3 供电不足 | 确保 ESP32 稳定 5V 供电，加 100uF 电容在 VIN 处 |
| 4 | 多个舵机同时动作 | 限制同时动作的舵机数量，或使用更大容量的 BEC |

## 二、电机问题 | Motor Issues

### 2.1 电机不转 / Motor Not Spinning

**症状 Symptom**：上电后电机完全无反应。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 驱动板未供电 | 检查 VM 端是否有电池电压（万用表测量） |
| 2 | UART 通信未连接 | 检查 Serial2 TX/RX 是否接对，TX→RX，RX→TX |
| 3 | 共地问题 | 驱动板 GND 必须与 ESP32-S3 GND 连接 |
| 4 | 电机模式未设置 | 确认代码中 `motors.setModes(4,4)` 被执行（模式 4 = 速度模式） |
| 5 | 电机三相线未连接 | 检查 A/B/C 三相线是否全部接好 |
| 6 | 驱动板故障 | 尝试单独给驱动板发送指令测试 |

### 2.2 电机转向错误 / Motor Runs in Wrong Direction

**症状 Symptom**：电机转动但方向相反。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 三相线顺序错误 | 交换任意两根相线（如 A↔B）即可反转 |
| 2 | 代码中方向标志反了 | 检查 `M0Dir` 和 `M1Dir` 变量。代码中默认为 -1，改为 1 可反转 |
| 3 | 仅一个电机反转 | 单独修改反转电机的 Dir 值 |

> 提示：代码中 `M0Dir = -1; M1Dir = -1;` 定义了电机方向。如果某个电机反转，将其对应的值改为 1（或反过来）。
>
> Tip: The code defines motor direction with `M0Dir = -1; M1Dir = -1;`. If a motor runs in reverse, change its Dir value to 1 (or vice versa).

### 2.3 电机抖动/异响 / Motor Jittering or Making Noise

**症状 Symptom**：电机发出嗡嗡声或抖动，不能平稳运转。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | PID 参数不当 | 调整电机速度环 PID 参数，降低 P 值 |
| 2 | 编码器信号异常 | 检查编码器连接线，确认无松动 |
| 3 | 供电电压不足 | 测量驱动板 VM 电压，确保 >10V |
| 4 | 机械卡死 | 检查轮毂是否与结构件摩擦 |
| 5 | 三相线接触不良 | 重新插紧三相线连接端子 |
| 6 | FOC 校准未完成 | 执行电机校准程序（`motors.alignSensors()`） |

### 2.4 电机过热 / Motor Overheating

**症状 Symptom**：电机壳体烫手（>60 度）。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 长时间堵转 | 检查是否有机械卡死，让电机自由运转 |
| 2 | 电流限制过高 | 降低电机驱动板电流限制 |
| 3 | 负载过重 | 检查机器人总重是否在电机额定范围内 |
| 4 | 散热不良 | 2808 电机为外转子设计，确保电机表面通风 |

## 三、IMU（惯性测量单元）问题 | IMU Issues

### 3.1 I2C 无法检测到 MPU6050 / MPU6050 Not Detected on I2C

**症状 Symptom**：运行 I2C 扫描程序后未发现 0x68 设备。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | SDA/SCL 接错 | 确认 SDA → GPIO 1, SCL → GPIO 2（不要接反） |
| 2 | I2C 地址不匹配 | 部分模块 AD0 默认拉高，地址为 0x69。试扫 0x69 |
| 3 | VCC 未连接 | 确认 MPU6050 VCC 接到 ESP32-S3 3.3V |
| 4 | 杜邦线接触不良 | 重新插拔杜邦线，或更换新的杜邦线 |
| 5 | 模块损坏 | 换一个 MPU6050 模块测试（价格便宜，建议备两个） |
| 6 | I2C 总线被其他设备拉低 | 断开 PCA9685，单独测试 MPU6050 |

### 3.2 姿态数据噪声大 / IMU Data Noisy

**症状 Symptom**：串口输出的 pitch/roll 角度值剧烈波动。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | IMU 模块松动 | 确保 MPU6050 紧固在框架上，不能有任何松动 |
| 2 | 电机振动传导 | 在 MPU6050 和框架之间加一块泡沫双面胶（减震） |
| 3 | I2C 信号受干扰 | 缩短 I2C 连接线长度（<10cm），或加 4.7k 上拉电阻 |
| 4 | 融合系数不当 | 调整代码中 `accCoef` 和 `gyroCoef`（默认 0.03 / 0.97） |
| 5 | 电源纹波 | 在 MPU6050 VCC 处加 0.1uF 陶瓷电容 |

### 3.3 姿态角度漂移 / IMU Angle Drift

**症状 Symptom**：机器人静止不动，但角度值持续缓慢变化。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 陀螺仪零偏未校准 | 运行 `mpu6050.calGyroOffsets()`，校准时**保持机器人完全静止** |
| 2 | 环境温度变化 | MPU6050 对温度敏感，预热 2-3 分钟后再校准 |
| 3 | 融合算法中 gyroCoef 过高 | 降低 gyroCoef（如从 0.97 降到 0.95），增加 accCoef |
| 4 | 电池电压变化 | 低电量时电压变化可能导致传感器偏移 |

> 校准提示：每次开机后，陀螺仪都会自动校准（代码中 `init()` 调用 `calGyroOffsets()`）。校准期间（约 4 秒）请保持机器人完全静止不动，否则零偏数据会不准确。
>
> Calibration tip: The gyroscope auto-calibrates on each boot (`init()` calls `calGyroOffsets()`). Keep the robot completely still during calibration (~4 seconds), otherwise the offset data will be inaccurate.

## 四、通信问题 | Communication Issues

### 4.1 SBUS 遥控器无法连接 / SBUS RC Controller Won't Connect

**症状 Symptom**：遥控器已开机但机器人无响应。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 接收机未对频 | 按照接收机说明书重新对频（bind） |
| 2 | SBUS 信号线接错 | 确认接收机 SBUS 信号 → ESP32-S3 GPIO 41 |
| 3 | 接收机供电不足 | 确认接收机有稳定 5V 供电 |
| 4 | Serial1 配置错误 | 确认代码中 `sbusRx.Begin(41, -1)` 正确执行 |
| 5 | 遥控器电池没电 | 更换遥控器 AA 电池或充电 |

### 4.2 蓝牙/WiFi 掉线 / Bluetooth/WiFi Disconnection

**症状 Symptom**：通过手机蓝牙/WiFi 控制时频繁断开。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 距离过远 | ESP32 蓝牙有效距离约 10m，WiFi 约 30m |
| 2 | 电池电量低 | 低电量时 ESP32 WiFi/蓝牙可能不稳定 |
| 3 | 天线被遮挡 | 确保 ESP32-S3 板载天线不被金属件遮挡 |
| 4 | 干扰源 | 避开 WiFi 路由器、微波炉等 2.4GHz 干扰源 |

### 4.3 ESP32-S3 无法烧录代码 / ESP32-S3 Upload Failure

**症状 Symptom**：PlatformIO 或 Arduino IDE 上传代码失败。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | COM 端口被占用 | 关闭串口监视器后重新上传 |
| 2 | USB 驱动未安装 | ESP32-S3 通常使用 CH340 或 CP2102 驱动，安装对应驱动 |
| 3 | 开发板型号选错 | PlatformIO 中选择 `esp32-s3-devkitc-1` |
| 4 | USB 线仅为充电线 | 使用带数据传输的 Type-C 线 |
| 5 | 自动复位失败 | 按住 BOOT 键，点击上传，出现 "Connecting..." 后松开 |

## 五、平衡控制问题 | Balance Issues

### 5.1 机器人立即倒下 / Robot Falls Immediately

**症状 Symptom**：上电后机器人无法站立，直接向前或向后倒下。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | MPU6050 安装方向错误 | 确认 IMU X 轴朝前，与代码坐标系一致 |
| 2 | 电机方向反了 | 修改 `M0Dir` 或 `M1Dir` 的正负号 |
| 3 | 左右电机接反 | 交换 M0 和 M1 的接线 |
| 4 | PID 参数偏差大 | 从 StackForce 课程代码的默认参数开始调试 |
| 5 | 电池电量不足 | 低电压时电机力矩不够，无法维持平衡 |
| 6 | 舵机未初始化到中位 | 确认上电后舵机自动回到 90 度中位 |
| 7 | 重心偏移过大 | 检查电池位置，调整到框架中心 |

### 5.2 机器人持续振荡 | Robot Oscillates Wildly

**症状 Symptom**：机器人能短暂站立但前后/左右剧烈摇摆。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 平衡 PID 的 P 值过大 | 降低 `kp1`（默认 0.38），每次减小 0.05 |
| 2 | 速度环 PID 参数不当 | 调整 `PID_VEL` 参数（默认 P=0.2, I=0, D=0） |
| 3 | IMU 数据噪声 | 参考 3.2 节降噪方案 |
| 4 | 机械松动 | 检查所有螺丝，特别是电机和舵机固定螺丝 |
| 5 | 控制频率不够高 | 确保主循环频率 > 100Hz |

### 5.3 机器人向一侧偏移 / Robot Drifts to One Side

**症状 Symptom**：机器人能基本平衡但持续向左或向右漂移。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 左右电机特性不一致 | 在代码中分别为 M0 和 M1 设置不同的扭矩系数 |
| 2 | 重心偏左/偏右 | 调整电池或电子元件位置使重心居中 |
| 3 | 舵机零点偏移 | 调整代码中的 `LFSERVO_OFFSET` 等偏移值 |
| 4 | 地面不平 | 在平整地面测试 |
| 5 | 轮子直径差异 | 检查两个轮子尺寸是否一致 |

## 六、3D 打印件问题 | 3D Print Issues

### 6.1 零件尺寸不合适/装不上 | Parts Don't Fit

**症状 Symptom**：3D 打印件无法与电机、螺丝或框架配合。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 打印机未校准 | 校准打印机的 XY 步进值和挤出流量 |
| 2 | 收缩/膨胀 | PLA 收缩率约 0.3%-0.5%，可在切片软件中设置缩放补偿 |
| 3 | 孔径偏小 | 用钻头或锉刀扩大安装孔（M3 孔扩到 3.2-3.4mm） |
| 4 | 层高过大导致尺寸偏差 | 使用 0.2mm 或更小层高重新打印 |

### 6.2 打印件层分离/开裂 | Layer Separation

**症状 Symptom**：打印件受力后沿层间开裂。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 打印温度过低 | PLA 建议喷嘴温度 200-210C |
| 2 | 层间冷却过快 | 关闭或降低零件冷却风扇（尤其前 3 层） |
| 3 | 打印速度过快 | 降低打印速度到 40-60mm/s |
| 4 | 材料受潮 | 烘干 PLA 耗材（50C 烘箱 4-6 小时） |

### 6.3 打印件翘曲 | Warping

**症状 Symptom**：打印件底部翘起，与热床分离。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 热床温度不够 | PLA 热床温度设为 60C |
| 2 | 无底筏（brim） | 在切片软件中添加 5-10mm 宽度的 brim |
| 3 | 环境温度低/有风 | 避免在空调出风口附近打印 |
| 4 | 热床未调平 | 重新调平热床，四角间隙一致 |

## 七、舵机问题 | Servo Issues

### 7.1 舵机不动作 / Servo Not Responding

**症状 Symptom**：舵机无反应，保持在当前位置。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | PCA9685 未检测到 | 运行 I2C 扫描，确认 0x40 存在 |
| 2 | OE 使能未拉高 | 确认 GPIO 42 输出 HIGH（`servos.enable()`） |
| 3 | 舵机电源未接 | 检查 PCA9685 V+ 端是否有 5V-6V |
| 4 | 舵机通道号错误 | 确认代码中通道号（CH3-CH6）与实际接线一致 |
| 5 | 舵机损坏 | 换一个已知正常的舵机测试 |

### 7.2 舵机抖动/发出嗡嗡声 | Servo Jittering

**症状 Symptom**：舵机在小幅振荡，发出嗡嗡声。

| # | 可能原因 | 解决方案 |
|---|---------|---------|
| 1 | 舵机负载过大 | 检查是否有机械卡死，减轻舵机负担 |
| 2 | PWM 频率不匹配 | 确认 PCA9685 频率设为 50Hz（代码中 `freq(50)`） |
| 3 | 脉冲范围设置不当 | 检查 `setPluseRange(500,2500)` 设置 |
| 4 | 舵机供电电压不稳 | 在 V+ 处加大电容（470uF-1000uF） |

## 调试三板斧 | The Three Debugging Steps

遇到任何硬件问题，按以下顺序排查：

> For any hardware issue, follow these steps in order:

### 1. 看串口 / Check Serial Output

打开串口监视器（921600 波特率），观察错误信息。这是最重要的调试手段。

> Open the serial monitor (921600 baud) and observe error messages. This is the most important debugging tool.

```cpp
// 在 setup() 中加入：
Serial.begin(921600);
Serial.println("System starting...");

// 在关键位置加入调试输出：
Serial.printf("MPU6050 angle: %.2f, %.2f\n", mpu6050.angle[0], mpu6050.angle[1]);
Serial.printf("Motor speed: M0=%.2f, M1=%.2f\n", BLDCData.M0_Vel, BLDCData.M1_Vel);
```

### 2. 查接线 / Check Wiring

用万用表逐一检查：

- **通断测试**：每条线的两端是否导通
- **电压测试**：各供电节点电压是否正确
  - 电池输出：11.1V-12.6V
  - 驱动板 VM：= 电池电压
  - ESP32-S3 VIN：5V（USB 供电时）
  - ESP32-S3 3.3V：~3.3V
  - MPU6050 VCC：~3.3V
  - PCA9685 V+：5V-6V

### 3. 隔离测试 / Isolate and Test

将问题隔离到最小系统：

- 只测 ESP32-S3 + MPU6050（确认 I2C 工作）
- 只测 ESP32-S3 + PCA9685 + 1 个舵机（确认舵机系统工作）
- 只测 ESP32-S3 + 驱动板 + 1 个电机（确认电机系统工作）
- 逐个加入其他组件，定位问题源头

## 常用备件建议 | Recommended Spare Parts

| 备件 | 数量 | 用途 |
|------|------|------|
| MPU6050 模块 | 1-2 个 | 最容易损坏的元件 |
| 杜邦线（母对母） | 1 包 | 接线容易折断 |
| M3 螺丝+螺母 | 1 包 | 容易丢失 |
| XT60 插头（公+母） | 1 对 | 电源插头 |
| 3S 锂电池 | 1 块 | 备用电池 |
| 舵机 | 1 个 | 备用舵机 |

> 建议每组额外准备上述备件，避免因小零件损坏导致整个项目停滞。
>
> Recommend each team prepare the above spare parts to avoid project delays due to minor component failures.
