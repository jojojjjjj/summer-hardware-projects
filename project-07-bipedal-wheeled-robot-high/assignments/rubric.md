# 评分细则 | Detailed Grading Rubric

## 总体说明 | Overall Description

本文档提供详细的评分标准，用于评估学员在轮足机器人项目中的表现。总分 100 分，共六个维度。

> This document provides detailed grading criteria for evaluating student performance in the bipedal wheeled robot project. Total score is 100 points across six dimensions.

---

## 评分维度总览 | Grading Dimensions Overview

| 维度 | 权重 | 满分 |
|------|------|------|
| 技术实现 Technical Implementation | 35% | 35 |
| 文档质量 Documentation | 10% | 10 |
| 演示展示 Presentation | 17% | 17 |
| 进度汇报 Check-ins | 15% | 15 |
| Demo 网站 Project Website | 13% | 13 |
| 团队协作 Collaboration | 10% | 10 |
| **总计 Total** | **100%** | **100** |

---

## 1. 技术实现 (35%) | Technical Implementation (35%)

### 1.1 平衡稳定性 (10%) | Balance Stability (10%)

| 等级 | 分数 | 描述 |
|------|------|------|
| **优秀 Excellent** | 9-10 分 | 机器人双轴自平衡非常稳定，能持续站立 1 分钟以上；抗干扰能力强，轻推后迅速恢复平衡；姿态角波动在 +/-2 度以内 |
| **良好 Good** | 7-8 分 | 机器人双轴自平衡较稳定，能持续站立 30 秒以上；有一定抗干扰能力；姿态角波动在 +/-3 度以内 |
| **中等 Average** | 5-6 分 | 机器人能自平衡但稳定性一般，能持续站立 10-30 秒；抗干扰能力较弱；姿态角波动在 +/-5 度以内 |
| **及格 Below Average** | 3-4 分 | 机器人能短暂自平衡（5-10 秒），但容易失去平衡；姿态角波动较大 |
| **不及格 Poor** | 0-2 分 | 机器人无法实现自平衡，或仅能单轴短暂平衡 |

### 1.2 遥控功能 (8%) | Remote Control Functionality (8%)

| 等级 | 分数 | 描述 |
|------|------|------|
| **优秀 Excellent** | 7-8 分 | 遥控功能完整（前进/后退/左转/右转/停止）；响应及时（<100ms）；运动平滑自然；有速度控制功能 |
| **良好 Good** | 6 分 | 遥控功能完整；响应较好（<200ms）；运动基本平滑 |
| **中等 Average** | 4-5 分 | 遥控基本功能可用（至少前进/后退/转弯）；响应有时延迟；运动偶有顿挫 |
| **及格 Below Average** | 2-3 分 | 遥控功能不完整（缺少部分方向）；响应延迟较大 |
| **不及格 Poor** | 0-1 分 | 遥控功能基本不可用或未实现 |

### 1.3 代码质量与结构 (7%) | Code Quality & Structure (7%)

| 等级 | 分数 | 描述 |
|------|------|------|
| **优秀 Excellent** | 6-7 分 | 代码模块化清晰（传感器/控制/电机/通信分离）；命名规范统一；有完整中文注释和 docstring；错误处理完善；有代码文档 |
| **良好 Good** | 5 分 | 代码有较好的模块化；命名基本规范；有关键代码注释；有基本错误处理 |
| **中等 Average** | 4 分 | 代码能运行但模块化一般；有部分注释；错误处理不足 |
| **及格 Below Average** | 2-3 分 | 代码能运行但结构混乱；注释稀少；无错误处理 |
| **不及格 Poor** | 0-1 分 | 代码无法编译运行；无注释；结构极差 |

### 1.4 PID 调参质量 (5%) | PID Tuning Quality (5%)

| 等级 | 分数 | 描述 |
|------|------|------|
| **优秀 Excellent** | 5 分 | PID 参数调优精确，系统响应迅速稳定；有完整的调参记录（参数表 + 响应曲线）；能清晰解释调参过程和原理 |
| **良好 Good** | 4 分 | PID 参数调优良好；有调参记录；能基本解释调参过程 |
| **中等 Average** | 3 分 | PID 参数基本可用；有简单调参记录 |
| **及格 Below Average** | 1-2 分 | PID 参数勉强可用；缺乏调参记录 |
| **不及格 Poor** | 0 分 | 未进行 PID 调参或参数极不合理 |

### 1.5 创新/附加功能 (5%) | Innovation / Extra Features (5%)

| 等级 | 分数 | 描述 |
|------|------|------|
| **优秀 Excellent** | 5 分 | 有 2 个以上创新附加功能（如视觉追踪、语音控制、自动巡线、手机 App 界面等）；功能完整且有实用价值 |
| **良好 Good** | 4 分 | 有 1-2 个附加功能；功能基本完整 |
| **中等 Average** | 3 分 | 有 1 个附加功能；功能可演示 |
| **及格 Below Average** | 1-2 分 | 有尝试添加附加功能但未完成 |
| **不及格 Poor** | 0 分 | 无附加功能 |

## 2. 文档质量 (10%) | Documentation (10%)

### 2.1 README 完整性 (3%) | README Completeness (3%)

| 等级 | 分数 | 描述 |
|------|------|------|
| **优秀 Excellent** | 3 分 | README 包含：项目简介、功能列表、硬件清单、软件架构、安装步骤、使用说明、故障排除、团队信息；中英双语 |
| **良好 Good** | 2.5 分 | README 包含大部分内容；信息较完整 |
| **中等 Average** | 2 分 | README 包含基本信息；部分内容缺失 |
| **及格 Below Average** | 1 分 | README 内容简陋；缺少关键信息 |
| **不及格 Poor** | 0 分 | 无 README 或 README 为空 |

### 2.2 代码注释 (3%) | Code Comments (3%)

| 等级 | 分数 | 描述 |
|------|------|------|
| **优秀 Excellent** | 3 分 | 每个模块有中文文档字符串；函数有参数和返回值说明；关键算法（PID、滤波）有详细注释 |
| **良好 Good** | 2.5 分 | 模块和函数有基本注释；关键部分有说明 |
| **中等 Average** | 2 分 | 有部分注释；覆盖不完整 |
| **及格 Below Average** | 1 分 | 注释稀少；仅有零星说明 |
| **不及格 Poor** | 0 分 | 无注释 |

### 2.3 接线图与原理说明 (4%) | Wiring Diagrams & Principle Explanations (4%)

| 等级 | 分数 | 描述 |
|------|------|------|
| **优秀 Excellent** | 4 分 | 有清晰的接线图（手绘或软件绘制均可）；有 PID 控制原理说明；有系统工作原理文档；有数据流图 |
| **良好 Good** | 3 分 | 有接线图；有基本的原理说明 |
| **中等 Average** | 2 分 | 有简单接线说明；原理说明不够清晰 |
| **及格 Below Average** | 1 分 | 接线说明不完整；缺少原理说明 |
| **不及格 Poor** | 0 分 | 无接线图和原理说明 |

## 评分流程 | Grading Process

### 1. 自我评估 (Day 12 上午)

学员填写自评表：
- 功能完成情况
- 代码质量评估
- 文档完整性
- 遇到的问题

### 2. 功能测试 (Day 12 下午)

现场验证：
- 自平衡稳定性测试
- 遥控功能测试
- 系统稳定性测试

### 3. 代码审查 (Day 12 下午)

检查：
- 代码质量与结构
- 注释完整性
- Git 提交历史
- 模块化程度

### 4. 演示评分 (Day 12 下午)

评估：
- 演示流畅度
- 讲解清晰度
- PPT 质量
- 问答表现

### 5. Demo 网站评估 (Day 12 下午)

检查：
- 网站已部署
- 内容完整性
- 视觉设计
- 技术深度

### 6. 综合评分 (Day 12 下午)

汇总各项得分，给出最终成绩

## 申诉流程 | Appeal Process

如果对成绩有异议：

1. **书面申诉** (3 天内)
   - 说明申诉理由
   - 提供证据材料

2. **复查评估** (5 天内)
   - 重新审查项目
   - 给出复查结果

3. **最终裁定** (7 天内)
   - 给出最终成绩
