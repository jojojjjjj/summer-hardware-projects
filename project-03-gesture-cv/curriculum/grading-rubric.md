# 评分标准 | Grading Rubric

> **手势互动装置项目** - 详细评分细则
>
> **Gesture Interactive Installation Project** - Detailed Grading Criteria

---

## 总体评分 | Overall Grading

| 类别 | 权重 | 满分 |
|------|------|------|
| 技术实现 | 40% | 40分 |
| 文档质量 | 20% | 20分 |
| 演示展示 | 20% | 20分 |
| 进度汇报 | 10% | 10分 |
| 团队协作 | 10% | 10分 |
| **总分** | **100%** | **100分** |

---

## 1. 技术实现 (40分) | Technical Implementation

### 功能完整性 (15分)
- [ ] 基础手势识别（5种） (5分)
- [ ] 音量控制功能 (5分)
- [ ] 虚拟画板功能 (5分)

### 代码质量 (15分)
- [ ] 代码结构和模块化 (5分)
- [ ] 代码可读性和命名规范 (5分)
- [ ] 错误处理和边界情况 (5分)

### 创新性 (10分)
- [ ] 自定义手势或功能 (5分)
- [ ] 硬件集成或创意扩展 (5分)

---

## 2. 文档质量 (20分) | Documentation Quality

### README文档 (8分)
- [ ] 项目介绍清晰 (2分)
- [ ] 安装说明完整 (2分)
- [ ] 使用说明详细 (2分)
- [ ] 示例和截图 (2分)

### 代码注释 (6分)
- [ ] 关键函数有注释 (2分)
- [ ] 复杂逻辑有解释 (2分)
- [ ] 中英双语注释 (2分)

### 演示PPT (6分)
- [ ] 内容完整 (2分)
- [ ] 设计美观 (2分)
- [ ] 时间控制合理 (2分)

---

## 3. 演示展示 (20分) | Presentation

### 演示效果 (10分)
- [ ] 功能演示流畅 (5分)
- [ ] 现场互动良好 (5分)

### 表达能力 (5分)
- [ ] 讲解清晰 (3分)
- [ ] 回答问题准确 (2分)

### 准备充分度 (5分)
- [ ] 设备测试完成 (2分)
- [ ] 备用方案准备 (2分)
- [ ] 时间控制 (1分)

---

## 4. 进度汇报 (10分) | Progress Reports

### Week 1 检查 (5分)
- [ ] 按时提交 (2分)
- [ ] 内容详实 (2分)
- [ ] 问题分析深入 (1分)

### Week 2 检查 (5分)
- [ ] 按时提交 (2分)
- [ ] 功能展示完整 (2分)
- [ ] 改进计划合理 (1分)

---

## 5. 团队协作 (10分) | Team Collaboration

### Git使用 (5分)
- [ ] 提交记录规范 (2分)
- [ ] Commit message清晰 (2分)
- [ ] 代码版本管理良好 (1分)

### 问题解决 (5分)
- [ ] 积极寻求帮助 (2分)
- [ ] 帮助他人 (2分)
- [ ] 参与讨论 (1分)

---

## 加分项 | Bonus Points

| 项目 | 分值 | 说明 |
|------|------|------|
| 额外功能 | +5 | 实现3个以上额外功能 |
| 硬件集成 | +5 | 成功集成LED或舵机 |
| 创新应用 | +5 | 独特的创意应用场景 |
| 优秀文档 | +3 | 文档质量超出预期 |
| 早期完成 | +2 | 提前完成所有要求 |

**最高加分:** 20分

---

## 扣分项 | Deductions

| 项目 | 分值 | 说明 |
|------|------|------|
| 延期提交 | -2/天 | 每天扣2分 |
| 代码抄袭 | -40 | 直接复制他人代码 |
| 缺席展示 | -20 | 无故缺席最终展示 |
| 文档缺失 | -5 | 缺少必需文档 |
