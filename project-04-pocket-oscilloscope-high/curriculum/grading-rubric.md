# 详细评分标准 | Detailed Grading Rubric

## 评分总览 | Grading Overview

| 评估维度 Dimension | 权重 Weight | 评分人 Evaluator |
|-------------------|-----------|-----------------|
| 技术实现 Technical Implementation | 35% | 导师 |
| 文档质量 Documentation | 9% | 导师 |
| 演示展示 Presentation | 17% | 导师 + 评委 |
| 进度汇报 Check-ins | 17% | 导师 |
| 团队协作 Collaboration | 9% | 导师 |
| Demo 网站 Project Demo Website | 13% | 导师 + 评委 |

---

## 1. 技术实现 (35%) | Technical Implementation

### 1.1 FPGA 设计 (10%) | FPGA Design (10 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 9-10 (优秀 Excellent) | DDS 核心完整实现（相位累加器 + 波形 LUT + 频率控制字）；支持多种波形输出（正弦/方波/三角波/锯齿波）；时序约束合理；仿真波形验证通过 |
| 7-8 (良好 Good) | DDS 基本功能实现，至少输出正弦波和方波；有时序约束；仿真验证基本正确 |
| 5-6 (及格 Pass) | 有 DDS 模块设计，但波形种类不足或仿真结果有偏差 |
| 3-4 (需改进 Needs Work) | FPGA 模块部分完成，DDS 未完全实现 |
| 0-2 (未完成 Incomplete) | FPGA 设计未完成或无法综合 |

### 1.2 ARM 嵌入式软件 (10%) | ARM Embedded Software (10 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 9-10 (优秀) | PS 端程序完整：AXI 通信正常、能动态配置 DDS 参数、LVGL GUI 交互流畅、触摸响应灵敏、异常处理完善 |
| 7-8 (良好) | PS-PL 通信正常，GUI 基本功能可用，能设置波形参数 |
| 5-6 (及格) | PS 端能运行，AXI 读写基本工作，但 GUI 不够完善 |
| 3-4 (需改进) | PS 端程序能编译运行，但 AXI 通信或 GUI 有明显问题 |
| 0-2 (未完成) | ARM 软件未完成或无法运行 |

### 1.3 系统集成 (10%) | System Integration (10 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 9-10 (优秀) | PS 与 PL 协同工作正常；Bitstream 加载 + PS 程序运行流畅；从 GUI 设置到波形输出的全链路打通；系统上电即可正常工作 |
| 7-8 (良好) | 系统能正常工作，偶尔需要重启；全链路基本打通 |
| 5-6 (及格) | 系统能工作，但需要手动干预（如多次加载）；部分功能不稳定 |
| 3-4 (需改进) | 系统部分集成，但核心链路未完全打通 |
| 0-2 (未完成) | 系统未能集成或无法运行 |

### 1.4 模拟输出与校准 (5%) | Analog Output & Calibration (5 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 5 (优秀) | 模拟输出波形清晰，无明显失真；频率精度 < 1%；幅度精度 < 5%；DAC8562 偏置控制正常 |
| 4 (良好) | 输出波形基本正常，频率/幅度精度在可接受范围内 |
| 3 (及格) | 能输出波形，但精度不够或有轻微失真 |
| 1-2 (需改进) | 输出波形失真严重或幅度不稳定 |
| 0 (未完成) | 模拟输出未工作 |

## 2. 文档质量 (9%) | Documentation Quality

### 2.1 README 与项目文档 (3 分) | README & Project Docs (3 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 3 (优秀) | README 完整规范（含项目说明、ZYNQ 架构图、使用方法、PS/PL 分工说明）；有系统框图和信号流图 |
| 2 (良好) | README 基本完整；有基本说明文档 |
| 1 (及格) | 有 README，但内容不够完整 |
| 0 (未完成) | 无 README 或几乎没有文档 |

### 2.2 代码注释 (3 分) | Code Comments (3 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 3 (优秀) | Verilog 模块有清晰的功能说明和端口注释；C 代码有函数文档和关键逻辑注释；寄存器映射有文档 |
| 2 (良好) | 代码有注释，覆盖主要模块 |
| 1 (及格) | 代码有部分注释 |
| 0 (未完成) | 代码几乎没有注释 |

### 2.3 原理图与架构图 (3 分) | Schematics & Architecture Diagrams (3 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 3 (优秀) | 有完整的系统架构图（PS/PL 分工）、信号链路框图（DDS -> DAC -> 运放 -> 输出）、AXI 地址映射表 |
| 2 (良好) | 有基本架构图和信号流说明 |
| 1 (及格) | 有简单框图 |
| 0 (未完成) | 无图或几乎无说明 |

## 3. 演示展示 (17%) | Presentation

### 3.1 现场演示 (8 分) | Live Demo (8 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 7-8 (优秀) | 现场演示流畅，能输出多种波形（正弦/方波/三角波/锯齿波），频率和幅度可调；用示波器实时验证波形参数；触摸屏操作顺畅 |
| 5-6 (良好) | 演示基本成功，能输出至少两种波形，操作偶有卡顿 |
| 3-4 (及格) | 能演示基本功能，但需要多次尝试或部分功能缺失 |
| 0-2 (需改进) | 演示不成功或无法正常工作 |

### 3.2 技术深度 (5 分) | Technical Depth (5 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 5 (优秀) | 能清楚解释 DDS 工作原理（相位累加器 -> LUT -> DAC）；理解 ZYNQ PS/PL 通信机制（AXI 总线）；能说明模拟信号链路设计思路 |
| 3-4 (良好) | 能解释主要技术原理；对部分细节理解不够深入 |
| 1-2 (及格) | 有基本理解，但回答问题需要提示 |
| 0 (需改进) | 无法解释核心技术原理 |

### 3.3 答疑能力 (4 分) | Q&A Ability (4 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 4 (优秀) | 回答评委提问准确、有条理；能对设计方案进行合理辩护；对技术取舍有清晰认识 |
| 3 (良好) | 回答基本正确，偶有不确定之处 |
| 2 (及格) | 能回答简单问题，复杂问题需要提示 |
| 0-1 (需改进) | 无法回答基本技术问题 |

## 5. 团队协作 (9%) | Collaboration

| 等级 Grade | 分数范围 Score Range | 说明 Description |
|-----------|-------------------|-----------------|
| A+ (卓越 Outstanding) | 95-100 | 远超预期，有创造性拓展（如自定义 ARB 波形、网络控制等） |
