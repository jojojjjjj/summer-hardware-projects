# HW1: DDS 原理理解与 Vivado 工程搭建 | DDS Principles & Vivado Project Setup

> **截止日期**：Day 2 结束时 | **权重**：12% | **预计耗时**：课后 2-3 小时

## 学习成果 | Learning Outcomes

完成本作业后，你将能够：

1. **解释** DDS（直接数字频率合成）的核心原理：相位累加器、波形查找表、频率控制字之间的关系
2. **识别** ZYNQ SoC 的 PS（ARM）和 PL（FPGA）分工协作模式，以及 AXI 总线在两者之间的桥梁作用
3. **实现** 一个完整的 Vivado ZYNQ Block Design 工程，包含 PS 配置和 PL 侧 GPIO 模块
4. **分析** 为什么 TinyAWG 选择 ZYNQ（ARM+FPGA）架构而非纯 MCU 方案

## 为什么学这个 | Why This Matters

Vivado 工程和 ZYNQ Block Design 是整个 TinyAWG 的**数字地基**。后续所有的 FPGA 逻辑（DDS 核心、AXI 寄存器）和 ARM 软件（驱动、GUI）都要在这个地基上搭建。

**日常类比**：想象你在盖一座桥。HW1 做的事情相当于"打桥墩+铺桥面"。PS 就像桥的南岸（管理交通调度），PL 就像桥的北岸（执行重型运输），AXI 总线就是连接两岸的桥面。桥面没铺好，两岸的人再忙也没法协作。

## 理解验证问题 | Comprehension Check

**Q1（设计权衡题）**：DDS 频率分辨率与累加器位宽的设计权衡

如果改用 16 位累加器，频率分辨率会变成多少？这会带来什么问题？在什么场景下 16 位累加器够用？

> **参考答案**：16 位累加器频率分辨率 ≈ 3052Hz。1kHz 输出时只能选择 0Hz 或 3052Hz，低频完全无法控制。但在只输出 10MHz 以上射频信号的场景可能够用。这体现了**精度-资源权衡**。

**Q2（原理追问题）**：PS 和 PL 为什么要用 AXI 总线而不是直接连线

如果 PS 直接连线到 PL 的每一个触发器，会有什么问题？

> **参考答案**：AXI 总线有三个关键作用：1) 地址映射——PL 寄存器映射到 PS 地址空间；2) 协议标准化——VALID/READY 握手；3) 可扩展性——添加新模块只需分配新地址。直接连线需要为每个寄存器分配物理引脚，引脚数量不够。类比：AXI 像大楼里的内部快递系统，写对门牌号就能送到。

**Q3（原理追问题）**：Block Design 中 Run Block Automation 做了什么

请解释 Run Block Automation 和 Run Connection Automation 分别做了什么？

> **参考答案**：Run Block Automation 自动完成 PS 的基础配置（DDR、FIXED_IO、时钟等）；Run Connection Automation 自动完成 IP 间连线（AXI 主从、时钟、复位）和地址分配。

**Q4（原理追问题）**：为什么 TinyAWG 带宽（35MHz）小于采样率的一半（100MHz）

请分析造成这个差距的因素有哪些？

> **参考答案**：1) 重建滤波器过渡带限制；2) DAC 零阶保持效应导致高频衰减；3) 运放高频增益下降；4) PCB 寄生参数影响。

**Q5（设计权衡题）**：AXI GPIO 与直接 Verilog 模块的取舍

什么时候应该用现成的 AXI GPIO IP，什么时候需要自己写自定义 IP？

> **参考答案**：AXI GPIO 适合简单 bit 控制（LED、按键），配置方便但灵活性有限。自定义 AXI Lite IP 适合复杂控制逻辑（DDS 需要多寄存器协同工作）。类比：AXI GPIO 像买现成家具，自定义 IP 像定制家具。

## 理解验证问题 | Comprehension Check

**Q1（原理追问题）**：相位累加器溢出与输出频率的关系

为什么"溢出"就等于"完成一个周期"？如果 FCW = 0，输出是什么？

> **参考答案**：LUT 地址取累加器高位。累加器从 0 增长到 2³²-1 再溢出回 0 时，高位完成 0→2ᴺ-1→0 循环，对应 LUT 中一个完整波形周期。FCW = 0 时累加器不变，LUT 地址固定，DAC 输出恒定直流值（0 Hz）。

**Q2（设计权衡题）**：查找表深度与频率分辨率的权衡

为什么不直接用 32 位地址的查找表？1024 点够用吗？

> **参考答案**：32 位查找表需要 56 GB 存储，FPGA 不可能实现。1024 点对低频信号足够（每周期大量采样点），但高频信号采样点少会导致失真。需要更高波形质量可增加 LUT 到 4096 或 8192 点，但消耗更多 BRAM。

**Q3（设计权衡题）**：多波形查找表的实现策略

为每种波形各建一个 BRAM，还是用一个 BRAM 的不同区域？分析优缺点。

> **参考答案**：方案 1（多 BRAM）切换快但消耗 4 倍资源。方案 2（分区 BRAM）节省资源，切换有 1-2 周期延迟但完全可接受。TinyAWG 推荐方案 2，因为 BRAM 资源有限且延迟对人眼无感。

**Q4（设计权衡题）**：AXI Lite 与 AXI Full 的选择

为什么 TinyAWG 用 AXI Lite 就够用？什么场景必须用 AXI Full？

**Q5（原理追问题）**：Verilog 阻塞赋值与非阻塞赋值

在时序逻辑 lways @(posedge clk) 中用阻塞赋值 = 会导致什么问题？

> **参考答案**：阻塞赋值立即生效，执行顺序依赖代码书写顺序，导致仿真和综合结果不一致。仿真可能正常（按代码顺序执行），但综合后触发器并行更新，行为不同。例： = b; b = a; 阻塞赋值时结果依赖顺序；非阻塞  <= b; b <= a; 正确交换值。

## 理解验证问题 | Comprehension Check

**Q1（原理追问题）**：双 DAC 架构的设计意图

为什么不用一颗 DAC 完成所有功能？如果只用 AD9744，怎么实现幅度调节？

> **参考答案**：只用 AD9744 时幅度调节需数字乘法，14 位乘法引入量化噪声降低有效位数。偏移控制更难——需在差分电流输出上叠加直流值。独立的 DAC8562 通过模拟方式（运放增益/偏移）实现更纯净的调节，避免数字量化精度损失。这是**数字便利性 vs 模拟精度**的权衡。

**Q2（原理追问题）**：14 位二进制补码数据格式

为什么不使用无符号格式？二进制补码在硬件实现中有什么优势？

> **参考答案**：补码优势：1) 正负对称——0x0000 代表 0V，幅度完全对称；2) 硬件简单——加减法统一，无需额外处理符号位；3) 与 DDS 自然对应——LUT 存储以 0 为中心的正弦波数据，直接输出即可。

**Q3（故障分析题）**：SPI 时序对 DAC8562 的影响

SPI 时钟设得太快或太慢分别会导致什么问题？

> **参考答案**：太快超过 DAC8562 最大 SPI 时钟（50MHz）会导致采样错误；太慢增加更新延迟但幅度/偏移控制可接受。推荐 10-20MHz。

**Q4（原理追问题）**：FCW 量化误差分析

要输出精确 1kHz，理论 FCW 是多少？实际 FCW 是多少？频率误差是多少 ppm？

> **参考答案**：理论 FCW = 1000 × 2³² / 200000000 = 21474.83648；实际取整 = 21475 (0x53E3)；实际频率 = 1000.00745 Hz；误差 ≈ 7.45 ppm，极小。

**Q5（设计权衡题）**：继电器输出保护的设计理由

为什么不用 FPGA 的 enable 信号直接控制，而用继电器？

> **参考答案**：FPGA enable 让 DAC 输出归零但运放仍在工作，有残余偏移和噪声。继电器提供**物理断开**——输出端与负载完全隔离，无残余信号。代价是切换慢（ms 级）且有机械寿命限制。类比：enable 像调音量到零，继电器像拔掉音箱线。

**Q1（设计权衡题）**：焊接顺序的设计理由

为什么必须按"电源→信号→数字"顺序？如果先焊数字再焊电源，可能遇到什么问题？

> **参考答案**：先焊数字的问题：1) 焊电源时高温可能损伤已焊好的数字芯片；2) 翻动 PCB 时已焊芯片可能被碰撞损坏；3) 最关键的是——先焊电源可以**先验证电源正常**再焊昂贵的信号芯片。如果电源有设计错误，可以在焊信号芯片前发现，避免损坏 AD9744。

**Q2（故障分析题）**：电源纹波对模拟信号的影响

如果 ±6.5V 电源纹波有 100mV，会对输出产生什么影响？

> **参考答案**：AD9744 的 14 位分辨率约 0.6mV/LSB。100mV 纹波相当于约 170 个 LSB 噪声，严重降低有效位数，使 14 位 DAC 实际性能只相当于 10-11 位。高速 DAC 对纹波特别敏感，因为内部参考电压受电源影响，且高速开关电流产生巨大瞬态电流。

**Q3（原理追问题）**：虚焊与冷焊的区别

请解释虚焊和冷焊的区别、形成原因和修复方法。

> **参考答案**：虚焊——焊锡看似连接但有氧化膜隔开，连接不稳定。外观暗淡粗糙。原因：温度不够或加热太短。修复：重新加热加助焊剂。冷焊——凝固过程中被振动导致内部裂纹。外观有细小裂纹。原因：未凝固时移动 PCB。修复：重新加热加新焊锡保持不动。两者共同危险：时好时坏，最难排查。

**Q4（故障分析题）**：QFN 芯片底部散热焊盘

如果焊接时没有把底部散热焊盘焊上，会导致什么问题？

> **参考答案**：1) 散热问题——QFN 通过底部焊盘散热，不焊会导致过热；2) 电气问题——底部焊盘常连接 GND，不焊可能缺少地线连接；3) 机械问题——底部焊盘提供主要固定力，不焊容易脱落。

**Q5（类比迁移题）**：系统调试的分层方法论

用一个日常类比来解释"先验证电源，再验证数字，最后验证模拟"。如果跳过电源验证直接调试模拟输出，最坏情况会怎样？

> **参考答案**：类比：修无法启动的汽车，先检查汽油（电源），再查火花塞（数字），最后查排气管（模拟）。跳过电源验证最坏情况：电源电压不对导致 AD9744 和 OPA2673 过压烧毁（损失约 45 元），甚至损坏 FPGA/ARM 核心板（损失约 60 元）。分层验证的核心价值：**在每一步排除一类问题，避免损坏硬件**。

## 理解验证问题 | Comprehension Check

**Q1（原理追问题）**：校准的必要性与限流因素

除了 FCW 量化误差外，还有哪些因素导致实际频率偏差？哪些可通过软件补偿？

> **参考答案**：可补偿：1) FPGA 时钟频率偏差——用实际测量时钟频率替换标称值；2) DAC8562 增益误差——查表补偿。不可补偿：1) 时钟抖动——随机波动无法校准消除；2) 温度漂移——只能在特定温度点校准；3) 运放带宽限制——物理特性无法软件修复。

**Q2（原理追问题）**：文档的受众分析

README 应写给谁看？"一个月后的你"和"不懂 FPGA 的同学"各关心什么？

> **参考答案**：一个月后的你最关心"为什么这样设计"——设计决策记录是最容易遗忘但最需要回忆的。不懂 FPGA 的同学最关心"一步一步怎么做"——任何步骤遗漏都可能导致失败。建议：开头放快速开始指南（新手），后面放技术细节和设计决策（有经验者）。

**Q3（故障分析题）**：项目可复现性测试

同学照着你的 README 做但遇到：1) Vivado 综合 module not found；2) 串口无输出；3) 波形有毛刺。请分析原因和解决方法。

> **参考答案**：1) Verilog 文件未添加到工程或模块名拼写错误——检查 Add Sources；2) UART 波特率不匹配或 Bitstream 未加载——确认 115200 和 JTAG 连接；3) 电源纹波、DAC 数据不同步、重建滤波器设置——按可能性从高到低排查。

**Q4（原理追问题）**：THD 与信号质量评估

THD 是怎么测量的？TinyAWG 输出 1kHz 正弦波时 THD 主要来自哪些因素？

> **参考答案**：THD = √(P₂+P₃+...) / P₁，通过 FFT 测量。理想正弦波 THD=0%。TinyAWG 的 THD 来源：LUT 量化、DAC 非线性（INL/DNL）、重建滤波器、运放非线性、电源纹波。专业 AWG THD < -60dB，TinyAWG 作为入门项目约 -40dB。

**Q5（设计权衡题）**：项目反思与设计改进

回顾整个项目，最大的三个技术风险是什么？如果重新设计你会做哪些改进？

> **参考答案**：1) ZYNQ 开发环境复杂度——可准备预配置工程模板；2) 模拟电路调试难度——可提供预焊模拟部分半成品；3) BRAM 资源限制——可升级 ZYNQ-7020 或使用压缩存储。重要的是展示**全局理解**和**权衡思维**。
