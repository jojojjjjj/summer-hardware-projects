# 硬件故障排除 | Hardware Troubleshooting

## 常见问题与解决方案 | Common Issues & Solutions

### 1. OLED 屏幕不亮 / 无显示

**症状 Symptom**：上电后 OLED 没有任何显示，完全黑屏。

**可能原因与解决**：
| # | 原因 | 解决 |
|---|------|------|
| 1 | 接线错误 | 检查 VCC→3.3V, GND→GND，不要接反 |
| 2 | 接触不良 | 重新插拔杜邦线，确保插紧 |
| 3 | SCL/SDA 接错 | 确认 SCL→GPIO0, SDA→GPIO2 |
| 4 | I2C 地址不匹配 | 大部分 SSD1306 地址是 0x3C，检查代码 |
| 5 | 屏幕损坏 | 换一块屏幕测试（概率极低） |

---

### 2. 烧录失败 / 上传不成功

**症状 Symptom**：Arduino IDE 上传代码时报错。

| 错误信息 | 原因 | 解决 |
|---------|------|------|
| `esptool.FatalError: Timed out waiting for packet header` | ESP 未进入烧录模式 | 检查烧录器开关是否拨到 PROG 位置 |
| `A fatal esptool.py error occurred` | COM 口被占用 | 关闭串口监视器，重新上传 |
| `Connecting..._____....._____` | 烧录器驱动问题 | 安装 CH340 驱动 |
| 未知 USB 设备 | 未装驱动 | 下载 CH340G 驱动并安装 |

**烧录操作正确步骤**：
1. 烧录器开关拨到 **PROG** 位置
2. 插入 USB
3. Arduino IDE 选择正确端口（COMx）
4. 选择开发板 "Generic ESP8266 Module"
5. 点击上传
6. 烧录完成后开关拨回 **UART** 位置
7. 按 RESET 或重新上电运行

---

### 3. WiFi 连接不上

**症状 Symptom**：屏幕一直显示"配网"或 WiFi 连接失败。

| # | 原因 | 解决 |
|---|------|------|
| 1 | WiFi 密码错误 | 重新进入配网模式，检查密码 |
| 2 | 2.4G/5G 问题 | ESP8266 仅支持 2.4G WiFi，不支持 5G |
| 3 | 路由器距离太远 | 靠近路由器测试 |
| 4 | 路由器 MAC 过滤 | 检查路由器设置 |
| 5 | wifi_link_tool 未正确配置 | 检查代码中 Hostname、rstb、stateled 引脚配置 |

---

### 4. 天气数据不显示 / 显示异常

**症状 Symptom**：OLED 能显示时间但天气区域为空或显示错误。

| # | 原因 | 解决 |
|---|------|------|
| 1 | API Key 无效 | 登录和风天气控制台确认 Key 状态 |
| 2 | 城市代码错误 | 检查 `dq` 变量，确认城市编号正确 |
| 3 | API 调用次数耗尽 | 免费版每天 1000 次，检查是否超额 |
| 4 | HTTPS 证书问题 | 检查 BearSSL 设置，确认 `client->setInsecure()` |
| 5 | JSON 解析失败 | 用串口监视器查看 API 返回内容 |

**调试方法**：
1. 打开 Arduino IDE 串口监视器（115200 波特率）
2. 查看 `接口返回` 后面的 JSON 数据
3. 将 JSON 复制到 https://json.cn 在线解析查看结构

---

### 5. 时间显示不正确

**症状 Symptom**：显示的时间与实际时间差几个小时。

| # | 原因 | 解决 |
|---|------|------|
| 1 | 时区设置错误 | 检查 `timeZone = 8`（中国标准时间 UTC+8） |
| 2 | NTP 服务器不通 | 修改 `timeServer` 地址尝试其他 NTP 服务器 |
| 3 | NTP 端口被封 | 有些网络封禁 123 端口（极少见），更换网络测试 |
| 4 | 首次同步未完成 | 上电后需要 1-2 秒才能同步，稍等一下 |

---

### 6. ESP-01S 发热严重

**症状 Symptom**：ESP-01S 模块摸起来很烫。

| # | 原因 | 解决 |
|---|------|------|
| 1 | 供电电压过高 | 万用表测量 VCC 是否为 3.3V，不要用 5V |
| 2 | 短路 | 检查面包板上是否有引脚短路 |
| 3 | 正常发热 | ESP8266 WiFi 工作时会发热，60-70°C 属于正常范围 |

---

## 调试三板斧 | The Three Debugging Steps

遇到任何问题，按以下顺序排查：

1. **看串口**：打开串口监视器看错误信息——这是最重要的调试手段
2. **查接线**：用万用表通断档检查每条线的连接
3. **简化测试**：烧录一个最简单的 Blink 程序，确认基础硬件正常
