# 最终展示要求 | Final Presentation Requirements

> 日期：Day 12 | Date: Day 12
> 形式：5-8 分钟口头演示 + 3 分钟 Q&A

---

## 演示结构 | Presentation Structure

### 1. 项目背景与目标 (1 分钟)
- 这个项目是什么
- 为什么要做这个项目
- 目标用户和使用场景

### 2. 技术架构概述 (1 分钟)
- 硬件方案（ESP8266 + OLED + 传感器）
- 软件架构（Arduino、API 调用、数据处理）
- 可以用一张架构图来说明

### 3. 硬件实物展示 + 功能演示 (2-3 分钟)
- 展示最终成品（外观、接线）
- 实时演示核心功能：
  - 时间显示是否正确
  - 天气数据是否正常获取
  - PM2.5 / B站粉丝数是否正常
  - WiFi 配网流程（可选演示）
- 如果条件允许，可用手机投屏展示 Demo 网站

### 4. 关键技术点讲解 (1-2 分钟)
- 选 1-2 个你认为最有价值的技术点深入讲
- 例如：HTTPS API 调用在 ESP8266 上的实现难点
- 例如：OLED 屏幕像素级 UI 布局设计

### 5. 遇到的问题与解决方案 (1 分钟)
- 选 1-2 个印象最深的问题
- 讲清楚：问题是什么 → 怎么发现的 → 如何解决的 → 学到了什么

---

## 演示技巧 | Presentation Tips

- **实物演示优先于 PPT**：观众更想看真实的设备运行
- **提前测试**：在演示场地提前测试 WiFi 和电源
- **准备备用方案**：如果现场网络不好，可以播放录制的视频
- **控制时间**：提前演练 2-3 遍，用手机计时
- **讲"为什么"**：不要只展示结果，要解释你为什么要这样设计
- **诚实面对问题**：如果功能不完美，坦诚说出原因和改进方向
