# 评分标准（进阶方向）| Grading Rubric (Advanced Track)

## 总体评分体系 | Overall Grading System

进阶方向采用六维度评分体系，与基础方向权重不同，更侧重于 **多模态 AI 集成** 和 **全栈开发能力**。

The advanced track uses a six-dimension grading system with different weights from the base track, emphasizing **multimodal AI integration** and **full-stack development skills**.

| 维度 | 权重 | 说明 |
|------|------|------|
| 技术实现 | 40% | ESP32 固件 15%、Python 后端 10%、AI 集成 10%、硬件组装 5% |
| 文档质量 | 15% | README、代码注释、架构图、API 文档 |
| 演示展示 | 15% | Final Demo 清晰度、表达能力、Q&A 应对 |
| Demo 网站 | 15% | 项目展示网站的可访问性、内容完整性、视觉呈现 |
| 进度汇报 | 10% | 每周 Check-in 质量、问题分析深度 |
| 创新拓展 | 10% | 额外功能、创意设计、个性化定制 |

**注意：** 百分比总和为 105%，这是有意设计 -- 创新拓展的 10% 作为额外加分项存在，实际总分上限仍为 100 分。

**Note:** The percentages sum to 105% by design -- the 10% Innovation serves as bonus credit. The maximum total score remains 100.

## 技术实现（40%）| Technical Implementation

### 细分权重 | Sub-weights

| 子维度 | 权重 | 说明 |
|--------|------|------|
| ESP32 固件（Arduino） | 15% | 固件功能完整度、代码质量、硬件驱动 |
| Python 后端 | 10% | API 服务稳定性、代码架构、错误处理 |
| AI 集成 | 10% | 多模态 AI 管线完整性（ASR + LLM + TTS） |
| 硬件组装 | 5% | 焊接质量、组装完整度、接线规范 |

### 评分细则

#### ESP32 固件（15%）| ESP32 Firmware

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | 所有硬件驱动正常工作，舵机控制精确流畅，屏幕动画丰富，摄像头流媒体稳定，代码结构清晰模块化 |
| 良好 | 80-89 | 核心硬件驱动正常，舵机和屏幕功能完整，代码可读性好，有少量不影响体验的问题 |
| 合格 | 70-79 | 主要功能实现，但缺少 1-2 个硬件驱动或有影响体验的问题 |
| 需改进 | 60-69 | 仅完成基本 LED 和串口功能，缺少屏幕/舵机/摄像头等核心驱动 |
| 不合格 | <60 | 固件无法编译或无法上传到 ESP32 |

| 类别 | 功能 | 分值 | 说明 |
|------|------|------|------|
| 基础 | Arduino IDE 编译上传正常 | 2 分 | 环境配置正确 |
| 基础 | GPIO/LED 控制 | 2 分 | LED 闪烁、按键输入 |

| 类别 | 功能 | 分值 | 说明 |
|------|------|------|

| 等级 | 分数范围 | 标准 |

| 不合格 | <60 | AI 对话功能无法运行 |

## 多模态 AI 功能专项评分 | Multimodal AI Feature Scoring

进阶方向的核心亮点是 **多模态 AI 集成**。以下是对 AI 功能的专项评分标准：

The core highlight of the advanced track is **multimodal AI integration**. Below are specific grading criteria for AI features:

### 语音识别（ASR）质量

| 等级 | 标准 |
|------|------|
| 优秀 | 能准确识别中英文混合语句，环境噪音下仍能工作，支持连续语音 |
| 良好 | 安静环境下识别准确，基本语句可识别 |
| 合格 | 能识别简单语句，但准确率一般 |
| 需改进 | 识别错误率高，经常需要重复 |

### 大模型对话（LLM）质量

| 等级 | 标准 |
|------|------|
| 优秀 | 回复连贯自然，有角色设定和上下文记忆，支持多轮复杂对话 |
| 良好 | 回复基本连贯，简单的上下文理解正常 |
| 合格 | 能产生回复但上下文理解较弱 |
| 需改进 | 回复不连贯或经常出错 |

### 语音合成（TTS）质量

| 等级 | 标准 |
|------|------|
| 优秀 | 语音自然流畅，语调有变化，支持不同角色音色 |
| 良好 | 语音清晰可懂，语速适中 |
| 合格 | 能播放语音但略显机械 |
| 需改进 | 语音不清晰或有明显杂音 |

### 端到端延迟

| 延迟 | 评价 |
|------|------|
| < 2 秒 | 优秀 -- 接近实时对话体验 |
| 2-5 秒 | 良好 -- 可接受的对话节奏 |
| 5-10 秒 | 合格 -- 稍有等待但可忍受 |
| > 10 秒 | 需改进 -- 影响用户体验 |
