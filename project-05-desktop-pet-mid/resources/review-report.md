## 三、适配性问题 | Adaptability Issues

### 问题 1（高优先级）：焊接难度对高中生的挑战

**问题描述：**
ESP32-S3-WROOM-1-N16R8 模组采用 QFN/LGA 封装，焊盘在芯片底部不可见，需要热风枪焊接。这对高中生来说是极大的挑战。QFN 焊接的典型故障模式：
- 锡膏过多 → 连锡短路 → 芯片烧毁
- 锡膏过少 → 虚焊 → 接触不良、间歇性故障
- 温度过高 → PCB 焊盘脱落 → 整块板报废
- 温度过低 → 焊接不充分 → 冷焊

**此外，其他贴片元件（0402/0603 封装阻容、ES8311 QFN-32、BMI270 LGA-14、FPC 座 0.5mm 间距）的焊接难度也不容小觑。**

**建议：**
1. 准备充足的废板和备用元件供练习（至少每人/组 2 块废板）
2. 准备 2-3 个备用 ESP32-S3 模组（焊接失败时的替代品）
3. 教师或助教需有 QFN 焊接经验，能在学员失败时帮助修复
4. 考虑提前预制 1-2 块焊接好的核心板作为"保底方案"，确保每个学员最终都有可用的硬件
5. 焊接前进行至少半天的废板练习（先用废旧主板/显卡练手）
6. 使用锡膏 + 热风枪方案（不要用电烙铁手工拖焊 QFN），降低难度

### 问题 2（高优先级）：安全风险管理

**问题描述：**
本项目涉及多种安全风险，必须有严格的培训和监督机制：

| 风险类别 | 具体风险 | 严重程度 |
|----------|----------|----------|
| 热风枪 | 温度 300-400°C，接触皮肤即烫伤 | 高 |
| 电烙铁 | 350°C，误触造成严重烫伤 | 高 |
| 铅烟 | 有铅锡膏/锡丝产生含铅烟雾，吸入有害 | 中 |
| 助焊剂烟雾 | 松香/助焊膏高温蒸发，刺激呼吸道 | 中 |
| 锂电池 | 过充/短路/物理损坏可能导致起火或爆炸 | 高 |
| 飞溅锡珠 | 焊接时锡珠飞溅可能进入眼睛 | 中 |
| 剪脚飞溅 | 水口钳剪排针时金属碎屑飞溅 | 低 |

**建议：**
1. 第一天必须进行 30 分钟以上的正式安全培训，含安全事故案例
2. 签署安全承诺书（学员和家长双签）
3. 焊接工位必须配备：排烟风扇、耐热垫、灭火器
4. 学员焊接时必须佩戴：KN95 口罩 + 护目镜（不佩戴不得焊接）
5. 锂电池充电必须有人看管，严禁无人充电过夜
6. 每个教室配备烫伤急救包（冷水、烫伤膏、纱布）
7. 明确规定：焊接完成后必须用肥皂洗手，工作台不能放食物饮料
8. 热风枪使用完毕必须放回支架，待冷却后再收纳

### 问题 3（中优先级）：ESP-IDF 学习曲线陡峭

**问题描述：**
ESP-IDF 是一个专业的嵌入式开发框架，对于高中生来说确实很陡峭：
- CMake 构建系统：与 Makefile/IDE 构建完全不同
- menuconfig 配置界面：终端文本界面，不直观
- 交叉编译工具链：xtensa-esp32s3-elf-gcc，与 PC 编译不同
- FreeRTOS：实时操作系统概念（任务、队列、信号量、互斥锁）
- 分区表、bootloader、OTA：嵌入式特有的概念
- C 语言指针/结构体回调：比 Python 复杂一个数量级

对于一个 12 天的课程来说，Day 1-4 的安排非常紧凑。如果学员在环境搭建上卡住，后续进度会受到严重影响。

**建议：**
1. Day 1 的 ESP-IDF 安装提供多套方案：
   - 方案 A：VS Code + ESP-IDF 插件（推荐）
   - 方案 B：命令行 `install.sh` / `install.bat` + VS Code
   - 方案 C：ESP Launchpad 在线烧录（备选，绕过编译环境）
2. 提前录制 ESP-IDF 安装视频，供学员回家提前安装（发放预习材料）
3. 教师提前在 U 盘上准备好离线安装包（ESP-IDF 完整安装包约 2-3GB）
4. 准备至少 2 台"黄金镜像"笔记本电脑，装有完整可用环境，供学员紧急使用
5. C 语言基础薄弱的学生建议提前 1-2 周自学（提供每日学习计划）

### 问题 4（中优先级）：3D 打印时间窗口紧张

**问题描述：**
SparkBot 外壳需要用光固化（SLA/DLP）树脂打印机打印，耗时通常为：
- 主体后壳：8-12 小时
- 前盖：4-6 小时
- 摄像头前后盖：2-3 小时
- 履带轮（×2）：4-6 小时

总打印时间约 18-27 小时（连续打印），但实际可能需要 2-5 天（因为有排队、清洗、固化等工序）。

如果在 Day 1 才开始打印，外壳最早在 Day 3 才能到——而此时学员已经开始焊接。最理想的情况是 Day 3-4 拿到外壳进行试装，但时间非常紧张。

**建议：**
1. 提前联系 3D 打印服务商，确认周期和数量
2. Day 1 当天（甚至 Day 0 报到日）就提交打印订单
3. 如果条件允许，学校自备 1-2 台光固化打印机（约 2000-3000 元/台）
4. 提前打印一套外壳作为展示样品（在学员打印到位前供参考）
5. 准备备用的简化版外壳方案（纸板/乐高/3D 打印笔手动制作）

### 问题 5（中优先级）：元器件采购风险

**问题描述：**
部分关键元器件可能缺货或断货：
- ESP32-S3-WROOM-1-N16R8：特定型号（必须 16MB Flash + 8MB PSRAM），容易与 N8R8/N8 版本混淆
- BMI270：Bosch 的 MEMS 传感器，供货不稳定
- 3.7V 2000mAh 特定尺寸电池：尺寸必须精确，否则放不进外壳

**建议：**
1. 提前至少 3 周采购所有元器件，预留补货时间
2. ESP32-S3 模组多买 30% 作为焊接损耗备品
3. 确认供应商的实际库存（不要仅看"有货"标签，直接联系客服确认）
4. BOM 清单中列出每个元器件的替代型号（如 BMI270 可用 BMI160 替代，需改代码）
5. 锂电池注意运输限制（部分快递不收锂电池），提前规划物流

### 问题 6（低优先级）：课程与代码架构的对齐

**问题描述：**
课程描述中提到了一些独立的驱动模块文件（如独立的 `sound_manager.c`、`particle_system.c`），但实际代码架构可能因整合优化而有所不同。这本身不是错误（模块整合是正常的软件工程实践），但可能让对照学习的学生产生困惑。

**建议：**
1. 在课程文档中添加一个"实际代码与课程示例的差异说明"部分
2. 明确标注哪些代码示例是教学版（独立的、利于理解），哪些是生产版（整合的、高效的）
3. 对于关键的驱动代码（如 BMI270、ST7789 初始化代码），保持课程描述与实际代码完全一致

### 问题 7（低优先级）：教师资质要求

**问题描述：**
本项目对教师的要求远高于 Python 桌宠项目：
- 需要嵌入式 C 编程能力（ESP-IDF 框架）
- 需要 QFN 焊接实操能力（能亲自示范和修复）
- 需要硬件调试能力（万用表、示波器、逻辑分析仪）
- 需要了解 FreeRTOS、I2C/SPI/I2S/DVP 等通信协议
- 需要 3D 打印知识（SLA 工艺、材料选择、后处理）
- 需要 AI 集成经验（LLM API、ASR/TTS）

**建议：**
1. 教师必须提前 1 个月完整复刻一遍 SparkBot（自己走完 Day 1-12）
2. 教师提前录制关键环节的视频教程（作为课堂补充）
3. 建议配备 1 名助教（研究生/高年级本科生），师生比不低于 1:8
4. 如果条件有限，可邀请当地创客空间/高校实验室提供技术支持

## 四、课程设计亮点 | Design Highlights

### 亮点 1：从"软件宠物"到"硬件机器人"的质变
学员不仅编写代码，还能亲手焊接、组装、调试一台真正的 AI 机器人。这种"从零到实物"的成就感远超纯软件项目。

### 亮点 2：技术栈覆盖全面
12 天的课程覆盖了嵌入式开发的完整技术栈：硬件（焊接/组装/3D打印）→ 驱动（I2C/SPI/I2S/DVP）→ 中间件（FreeRTOS/LVGL/ESP-WHO）→ 应用（天气/AI对话/游戏）→ 网络（WiFi/HTTP/蓝牙）

### 亮点 3：开源社区支持充分
ESP-SparkBot 有活跃的中文社区（Gitee 1K+ Stars、专门论坛、多个复刻教程），学员遇到问题有地方求助。

### 亮点 4：安全培训体系完善
课程纳入了正式的安全培训、防护装备清单、安全承诺书、急救预案，体现了对学员安全的高度重视。

### 亮点 5：真实工程体验
学员经历的不是"玩具项目"，而是真实的嵌入式产品开发流程——从采购到焊接、从驱动开发到系统集成、从调试到演示。

**修复建议：** 将 `curriculum/grading-rubric.md` 的权重统一为 README.md 和 `assignments/rubric.md` 的版本（文档 15%、演示 15%、新增 Demo 网站 15%）。

### 5.2 硬件项目评分特殊考量

与 Python 软件项目不同，ESP-SparkBot 的评分需要特别考量硬件因素：

1. **硬件不可抗力**：如果学员焊接的 ESP32-S3 模组确实无法正常工作（即使是焊接技能不错的学员也可能遇到），评分应在"功能完整度"维度给予适当宽容，重点评估代码质量和理解深度
2. **备件方案**：如果教师提供了预制的备件核心板，学员使用了备件板但代码是自己写的，评分应扣除焊接分但保留代码分
3. **3D 打印延迟**：如果外壳因打印服务商问题延迟到达，不应影响评分
