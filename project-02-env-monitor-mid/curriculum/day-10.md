## 📖 上午: 外壳设计原理 | Morning: Enclosure Design Principles

### 为什么需要外壳? | Why Need Enclosure?

**功能:**
- ✅ 保护电路和传感器
- ✅ 提供专业外观
- ✅ 便于安装和携带
- ✅ 优化传感器性能(如气流)

**设计考虑:**

| 因素 | 说明 | 示例 |
|------|------|------|
| **散热** | ESP32工作时发热 | 需要通风孔 |
| **气流** | PM传感器需要空气进出 | 进气/排气口设计 |
| **防水** | 防止液体溅入 | 密封设计 |
| **可维护** | 便于检修 | 可拆卸设计 |
| **美观** | 产品化外观 | 颜色/质感 |

> **Good Enclosure:** Protects electronics, optimizes sensor performance, looks professional

**为什么要校准?**  
传感器存在偏差，校准可以提高精度。

> **Why calibrate?** Sensors have deviations, calibration improves accuracy

**校准方法:**

**1. BME680温度校准:**
```python
# 与标准温度计对比

sensor_temp = 26.5    # BME680读数

# 在代码中应用
corrected_temp = raw_temp + offset
```

### 任务10.6: 功耗优化 (30分钟)

**测量功耗:**
```python
# 电流测量
# 使用万用表串联在电源回路
# 测量不同模式电流:

# 模式1: 正常运行
# 模式2: 深度睡眠
# 模式3: WiFi关闭

# 优化策略:
# 1. 降低CPU频率
machine.freq(160000000)  # 160MHz → 80MHz

# 2. 关闭不需要的外设
# 3. 使用深度睡眠
import machine
machine.deepsleep(60000)  # 睡眠60秒
```

**休眠模式实现:**
```python
import machine

# 配置唤醒源
# 1. 定时器唤醒
rtc = machine.RTC()
rtc.alarm(rtc.ALARM0, 60000)  # 60秒后唤醒

# 2. GPIO唤醒
pin = machine.Pin(0, machine.Pin.IN)
machine.wake_on_ext0(pin, machine.Pin.WAKE_HIGH)

# 进入深度睡眠
machine.deepsleep()
```
