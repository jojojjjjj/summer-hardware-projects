# 最终展示要求 | Final Presentation Requirements

## 概述 | Overview

在 Day 12，每位学员（或每组）将有 **8分钟演示 + 2分钟Q&A** 的时间，向全班展示你亲手打造的 ESP-SparkBot AI桌面机器人。

On Day 12, each student (or team) will have **8 minutes for demo + 2 minutes for Q&A** to present their hand-built ESP-SparkBot AI desktop robot to the class.

---

## 演示结构 | Presentation Structure

### Part 1: 项目介绍（1分钟）| Project Introduction

**必须涵盖的内容：**
- 你的 SparkBot 叫什么名字？有什么个性/特点？
- 什么是 ESP-SparkBot？简单一句话介绍（基于 ESP32-S3 的开源 AI 桌面机器人）
- 开源地址：https://gitee.com/esp-friends/esp_sparkbot
- 为什么选择这个项目？它最吸引你的地方是什么？

**可选加分：**
- 展示你的 SparkBot 定制化内容（自定义 UI 主题、自定义表情、自定义语音）

---

### Part 2: 硬件展示（2分钟）| Hardware Showcase

**必须展示的内容：**

1. **拼接完成的 PCB** -- 向观众展示你亲手焊接的三块 PCB（核心板、底板、履带底盘）
   - 指出各模块位置：ESP32-S3 模组、OV2640 摄像头、ST7789 屏幕、ES8311 音频芯片、BMI270 IMU
   - 展示焊点质量（近景特写）

2. **3D 打印外壳** -- 展示外壳的五部分（主体后壳、前盖、摄像头前盖、摄像头后盖、履带轮）
   - 讲述外壳是如何通过 3D 打印制作的
   - 展示组装完成后的外观

3. **组装过程回顾** -- 用 2-3 张照片/短视频回顾：
   - Day 2-3：焊接过程（烙铁、热风枪、锡浆、助焊膏）
   - Day 3-4：组装过程（核心板插入底板、摄像头安装、FPC 排线连接、外壳合盖）
   - Day 4：首次上电点亮的激动时刻

---

### Part 3: 功能演示（3分钟）| Live Feature Demo

按以下顺序进行现场演示（每人/组选择 4-5 个核心功能 + 1-2 个加分功能）：

**核心功能（至少演示 4 个）：**

1. **天气时钟** -- 展示 LCD 显示时间、日期、天气图标、温度
2. **表情动画** -- 展示 SparkBot 的表情切换（开心/睡眠/思考/眨眼）
3. **触摸交互** -- 触摸顶部/侧面触发不同表情或动作
4. **AI 对话** -- 对着 SparkBot 说一句话，展示 AI 大模型的语音回复
5. **木鱼/骰子小游戏** -- 展示触摸敲木鱼或摇骰子功能

**加分功能（有则演示）：**

6. **人脸识别** -- 摄像头检测人脸，LCD 显示检测框
7. **拍照功能** -- 拍照并保存，通过 WiFi 传输到手机浏览器查看
8. **蓝牙音箱** -- 手机连接 SparkBot 蓝牙，播放音乐
9. **履带移动** -- 底盘运动控制（前进/后退/转弯）
10. **动作感应** -- 拍手/摇晃触发的交互反应

**演示技巧：**
- 提前录好关键功能的演示视频作为现场备份（以防硬件故障）
- 备一个充电宝，确保 SparkBot 演示过程中不会断电
- 提前测试教室 WiFi，确保 AI 对话的 API 能连通

### Part 4: 技术总结（2分钟）| Technical Summary

选择以下 **一个** 话题深入讲解：

**话题 A：系统架构设计**

画出 SparkBot 的系统架构图，包括：
- 硬件层：ESP32-S3 主控 + 各外设模块（屏幕、摄像头、音频、IMU、WiFi/BLE）
- 驱动层：I2C（BMI270、ES8311）、SPI（ST7789）、DVP（OV2640）、I2S（音频）、GPIO（触摸）
- 中间件层：FreeRTOS（任务调度）、LVGL（GUI）、ESP-WHO（人脸识别）、ESP-SR（语音识别）
- 应用层：天气时钟、AI 对话、小游戏、蓝牙音箱、人脸识别

解释各层之间的数据流和通信方式。

**话题 B：一个技术难点的攻克过程**

选择以下一个难点详细展开：
- **ESP32-S3 模组焊接** -- QFN/LGA 封装的焊接方法、热风枪温度曲线、连锡/虚焊的排查和修复
- **I2C 设备调试** -- 当传感器不响应时的排查步骤（示波器看波形、上拉电阻检查、地址扫描、寄存器读取）
- **LVGL 内存优化** -- ESP32-S3 PSRAM 的使用、显示缓冲区的选择、帧率优化
- **音频流水线调试** -- I2S 时钟配置、ES8311 寄存器配置、麦克风偏置电压、降噪参数
- **FreeRTOS 任务优先级** -- 如何平衡 UI 刷新、音频处理、网络请求的 CPU 资源

**话题 C：代码架构与模块设计**

展示你的项目文件结构，并解释每个模块的职责：
```
main/
  ├── CMakeLists.txt
  ├── main.c              # 主入口：硬件初始化 + FreeRTOS 任务创建
  ├── app_weather.c/.h    # 天气时钟应用
  ├── app_ai_chat.c/.h    # AI 对话应用
  ├── app_game.c/.h       # 木鱼/骰子小游戏
  ├── app_bluetooth.c/.h  # 蓝牙 A2DP 应用
  ├── gui/
  │   ├── gui_main.c/.h   # LVGL 主界面管理
  │   ├── gui_weather.c   # 天气界面
  │   ├── gui_face.c      # 表情动画
  │   └── gui_menu.c      # 设置菜单
  ├── drivers/
  │   ├── bmi270.c/.h     # BMI270 IMU 驱动
  │   ├── st7789.c/.h     # ST7789 显示屏驱动
  │   ├── ov2640.c/.h     # OV2640 摄像头驱动
  │   ├── es8311.c/.h     # ES8311 音频驱动
  │   └── touch.c/.h      # 触摸驱动
  └── utils/
      ├── wifi.c/.h       # WiFi 管理
      ├── http_client.c   # HTTP 请求（天气 API / LLM API）

```

## Q&A 准备 | Q&A Preparation

以下是可能被问到的问题，提前准备好答案：

| 可能的问题 | 准备方向 |
|------------|----------|
| ESP32-S3 和普通的 ESP32 有什么区别？ | 双核 Xtensa LX7、支持 PSRAM、USB-OTG、更强的 AI 加速能力 |
| 你的状态机/任务调度是怎样的？ | FreeRTOS 有几个任务？优先级如何分配？任务间如何通信？ |
| I2C 和 SPI 有什么区别？为什么屏幕用 SPI 而不是 I2C？ | SPI 速度快（最高 80MHz vs I2C 400KHz），适合屏幕这种大数据量传输 |
| 为什么选择 ESP-IDF 而不是 Arduino？ | ESP-IDF 是官方框架，支持更多硬件功能（PSRAM、DVP、I2S）、更好的性能和调试能力 |
| FreeRTOS 任务之间如何通信？ | 队列（Queue）、信号量（Semaphore）、事件组（Event Group）、任务通知 |
| 如何解决 LVGL 的帧率问题？ | 使用 PSRAM 作为显示缓冲区、降低分辨率、使用 DMA 传输、选择合适的 `LV_MEM_SIZE` |
| 如果让你继续开发，你最想加什么功能？ | 准备 1-2 个拓展方向（本地语音唤醒、自训练人脸识别、手机 App 远程控制等） |
| 蓝牙和 WiFi 可以同时使用吗？ | 可以，ESP32-S3 支持 WiFi + BLE 共存（共存模式配置）|
| 为什么焊接 ESP32-S3 模组时不能用太多锡膏？ | QFN 封装焊盘在底部，锡膏过多会导致连锡短路 |
| 你的项目总共花费多少钱？总工时多少？ | 提前统计 BOM 总成本和代码行数 |

### 备用故障预案 | Emergency Backup Plan

如果现场演示出现硬件故障，请准备以下预案：

| 故障场景 | 预案 |
|----------|------|
| SparkBot 无法开机 | 使用提前录制的功能演示视频代替现场演示 |
| WiFi 无法连接（API 调用失败） | 展示离线功能（天气时钟、表情、小游戏、蓝牙），解释 API 调用原理通过代码讲解 |
| 屏幕不显示 | 通过串口日志展示程序在后台正常运行，同时用之前录制的屏幕视频代替 |
| 电池没电 | 使用 USB 线直接供电，同时展示充电中的指示灯 |
| 摄像头无画面 | 展示已保存的拍照照片，解释 DVP 接口和数据流原理 |

## 演示小贴士 | Presentation Tips

1. **讲故事** -- 不只是展示功能，讲述你从零基础到做出 AI 机器人的经历：第一次焊接时的紧张、首次上电点亮的兴奋、通宵调试 I2C 的挣扎
2. **互动** -- 邀请同学上来触摸 SparkBot、对它说话、玩木鱼小游戏，增加参与感
3. **真诚** -- 如果某个功能没完全做好，坦诚说明遇到的困难和分析的原因，比假装完美更打动人
4. **时间控制** -- 提前在宿舍练习 2 遍，确保不超过 8 分钟。录下自己练习的音频听一遍
5. **镜头意识** -- 演示硬件时动作慢一些，确保摄像头/投影能看清你的手部操作和 LCD 屏幕
6. **对比展示** -- 展示 Day 1 的"废板"练习焊点 vs 最终 SparkBot 的焊点，体现成长
7. **感谢环节** -- 最后感谢你的搭档、帮助过你的同学和老师
