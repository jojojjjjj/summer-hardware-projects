# 最终展示要求 | Final Presentation Requirements

## 概述 | Overview

在Day 12，每位学员将有 **8分钟演示 + 2分钟Q&A** 的时间，向全班展示你的桌宠作品。

On Day 12, each student will have **8 minutes for demo + 2 minutes for Q&A** to present their desktop pet to the class.

---

## 演示结构 | Presentation Structure

### Part 1: 开场介绍（1分钟）

- 你的桌宠叫什么名字？有什么个性/特点？
- 为什么选择这个名字/形象？
- 一句话概括你的桌宠最大的亮点

### Part 2: 功能演示（4分钟）

按以下顺序演示你的桌宠功能：

**必须演示的功能：**
1. 透明窗口显示 -- 桌宠"站"在桌面上
2. 帧动画 -- 至少展示2种动画状态的切换
3. 状态机自动行为 -- 展示桌宠自主行为（如自动走路、睡觉）
4. 鼠标交互 -- 点击、拖拽移动桌宠
5. 右键菜单 -- 展示菜单功能（喂食、查看状态等）
6. 气泡对话 -- 桌宠通过气泡表达想法

**加分功能（有则演示）：**
7. 音效和粒子特效
8. AI对话功能
9. 硬件传感器互动
10. 其他创新功能

### Part 3: 技术分享（2分钟）

选择以下一个话题深入讲解：

**话题A：状态机设计**
- 画出你的状态转换图
- 解释你的状态机有多少个状态
- 说明状态转换的条件

**话题B：技术难点攻克**
- 描述你遇到的最有挑战的技术问题
- 展示你的解决思路和过程
- 最终的解决方案

**话题C：代码架构**
- 展示你的项目文件结构
- 解释各个模块的职责
- 说明模块之间的通信方式

### Part 4: 学习反思（1分钟）

回答以下问题：
- 这12天你学到了什么最重要的东西？
- 如果重新开始，你会做哪些不同的设计？
- 接下来你想继续探索什么方向？

- [ ] 确认音量合适（如果演示音效）
- [ ] 准备好Q&A的回答

## 🌐 项目展示网站 | Project Demo Website

**这是最终评分的重要组成部分。** 你必须创建一个在线可访问的项目展示网站。

**This is a critical part of your final grade.** You must create an online-accessible project demo website.

### 要求 | Requirements

1. **部署平台** — 使用以下任一免费平台部署：
   - GitHub Pages（推荐）— 推送 HTML 到仓库的 `docs/` 分支或 `gh-pages` 分支
   - Vercel — 连接 GitHub 仓库自动部署
   - Netlify — 拖拽文件夹即可部署
   - 其他可公网访问的平台均可

2. **网站内容必须包含：**
   - [ ] 项目介绍（名称、目标、核心功能）
   - [ ] 桌宠展示（角色设计、动画截图/GIF、交互演示视频）
   - [ ] 软件架构（系统框图、状态机设计、模块说明）
   - [ ] 演示视频或 GIF（展示桌宠在桌面上的实际运行效果）
   - [ ] 技术亮点（状态机、动画系统、AI对话等）
   - [ ] 12天学习心得与总结
   - [ ] B站参考视频链接列表

3. **技术建议：**
   - 最简方案：一个 `index.html` 文件 + CSS 样式即可
   - 可使用 GitHub Pages 的 Jekyll 主题快速生成
   - 推荐工具：Markdown + 静态网站生成器（如 VitePress、Hugo）

### 提交方式 | Submission

- 在最终展示前，将网站 URL 提交给老师
- URL 格式示例：`https://你的用户名.github.io/project-name/`
- 老师将通过访问该 URL 进行评分

### 评分标准 | Scoring

| 维度 | 分值 | 说明 |
|------|------|------|
| 网站可访问性 | 5分 | URL可正常打开，无404错误 |
| 内容完整性 | 5分 | 包含所有要求的内容模块 |
| 视觉呈现 | 3分 | 页面整洁、排版合理、图文并茂 |
| 技术展示深度 | 2分 | 对核心技术的解释清晰准确 |
| **小计** | **15分** | |

## 演示小贴士 | Presentation Tips

1. **讲故事**：不只是展示功能，讲述你开发过程中的有趣经历
2. **互动**：邀请同学上来操作你的桌宠，增加参与感
3. **坦诚**：如果某个功能没完全做好，坦诚说明遇到的困难和学到的经验
4. **控制时间**：提前练习一遍，确保不超过8分钟
5. **准备备用方案**：如果现场演示出问题，准备好录屏或截图作为替代

## Q&A准备 | Q&A Preparation

以下是可能被问到的问题，提前准备好答案：

| 可能的问题 | 准备方向 |
|------------|----------|
| 你的状态机有多少个状态？ | 列出所有状态和转换条件 |
| 为什么选择PyQt5而不是Tkinter？ | 比较两者的优缺点 |
| 如何让桌宠的行为更自然？ | 讨论随机性和需求系统 |
| 代码有多少行？总共花了多少时间？ | 提前统计 |
| 如果让你继续开发，你最想加什么功能？ | 准备1-2个拓展方向 |
| 打包后的exe有多大？为什么？ | 解释PyQt5打包的原理 |
