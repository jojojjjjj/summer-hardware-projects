# 故障排除 | Troubleshooting

## 概述 | Overview

本文档汇总 ESP-SparkBot 在烧录、硬件和功能测试中可能遇到的常见问题及其解决方案。排查问题时，请按顺序执行每个检查步骤，不要跳过。

This document collects common issues encountered during flashing, hardware debugging, and functional testing of ESP-SparkBot, along with their solutions. When troubleshooting, follow each check step in order — do not skip.

---

## 1. 烧录问题 | Flashing Issues

### 1.1 电脑无法识别 COM 口

**症状**：插入 USB-C 线后，设备管理器中没有任何新串口出现。

| 排查步骤 | 操作 |
|----------|------|
| 1 | **按住 BOOT 键再插 USB** — 这是最常见的原因。ESP32-S3 首次烧录或固件异常时不会自动进入下载模式，必须手动按住 BOOT 键上电。松手时机：电脑识别到 COM 口后。 |
| 2 | 换一根 USB-C 数据线 — 很多 USB-C 线只能充电不能传数据。确认是**数据线**。 |
| 3 | 换一个 USB 口 — 避免使用 USB Hub（供电不足），直连电脑 USB 口。 |
| 4 | 检查设备管理器 — Windows 下看"端口(COM 和 LPT)"下是否有新设备。如果显示"未知设备"或感叹号，说明驱动未安装。 |
| 5 | 安装 CP210x 或 CH343 驱动 — ESP32-S3 开发板常用 CP2102/CH343 USB-UART 芯片。到 Silicon Labs 或 WCH 官网下载对应驱动。 |
| 6 | 万用表测 USB-C VBUS 是否有 5V — 如果没有，检查 Type-C 焊接（最有可能连锡短路或虚焊）。 |

### 1.2 烧录后无限重启 (rst:0x10)

**症状**：烧录完成后串口不断输出 `rst:0x10 (RTCWDT_RTC_RESET)` 或类似复位信息。

| 原因 | 解决方案 |
|------|----------|
| 分区表错误 | 检查 `partitions.csv` 是否匹配 Flash 大小 (16MB)。Flash 是 N16，分区表不能超过 16MB。 |
| 固件太大 | 检查编译输出的固件大小是否超过分区表的 app 分区大小。 |
| PSRAM 未使能 | 在 `menuconfig` 中开启 PSRAM (Component config → ESP PSRAM → Support for external, SPI-connected RAM)。模组是 N16**R8** (带 8MB PSRAM)，必须启用。 |
| Flash 模式错误 | 确认 Flash 模式为 QIO 或 DIO (menuconfig → Serial flasher config → Flash SPI mode)。 |
| 供电不足 | 电脑 USB 口供电不足导致 ESP32-S3 启动瞬间电压跌落。换一个 USB 口或外接 5V 电源。 |

**快速诊断命令**：
```bash
# 用 esptool 读取 Flash 信息
esptool.py --port COMx flash_id

# 如果读不到或报错，说明 Flash 焊接有问题或供电问题
```

### 1.3 编译找不到头文件

**症状**：`fatal error: xxx.h: No such file or directory`

| 原因 | 解决方案 |
|------|----------|
| 缺少组件依赖 | 检查 `CMakeLists.txt` 中的 `REQUIRES` 列表是否包含了该头文件所属的组件。例如使用 I2S 需要 `REQUIRES driver`，使用摄像头需要 `REQUIRES esp32-camera`。 |
| 组件路径不对 | 检查 `components/` 目录下是否有对应的组件文件夹。开源项目通常需要手动 `git clone` 组件仓库到 `components/`。 |
| ESP-IDF 版本不匹配 | 确认 ESP-IDF 版本与项目要求一致（通常 v5.0+）。不同版本 API 和头文件路径可能不同。 |

## 2. 显示问题 | Display Issues

### 2.1 屏幕只有背光亮，无任何显示

**症状**：上电后屏幕白色背光亮起，但没有图像内容。

| 排查步骤 | 操作 |
|----------|------|
| 1 | **检查 SPI 模式** — ST7789 必须使用 **SPI_MODE0**。在初始化代码中确认：`spi_device_interface_config_t.mode = 0`。 |
| 2 | 检查排线焊接 — ST7789 的 FPC 排线座焊点有无虚焊（用放大镜看每个引脚）。 |
| 3 | 检查排线方向 — FPC 排线是否插反（金属触点面朝向是否正确）。 |
| 4 | 用万用表蜂鸣档通断测试 — 从 ESP32-S3 的 SPI 引脚到 FPC 座对应引脚是否导通。 |
| 5 | 检查 CS 和 DC 引脚 — 这两个引脚如果接错，屏幕不会响应 SPI 数据。 |
| 6 | 换一块屏幕测试 — ST7789 屏幕本身偶尔有不良品，换一块排除。 |

### 2.2 屏幕花屏/颜色异常

**症状**：画面有内容但颜色不对、条纹、或画面偏移。

| 原因 | 解决方案 |
|------|----------|
| SPI 频率过高 | 降低 SPI 时钟频率（尝试从 40MHz 降到 20MHz 或 10MHz）。飞线或长排线易受干扰。 |
| 分辨率设置错误 | ST7789 240×240 的初始化序列中分辨率参数是否正确。 |
| 画面偏移 | 检查 `x_start` / `y_start` 偏移参数，部分 ST7789 面板起始行/列不为 0。 |
| Gamma 曲线 | 修改 ST7789 初始化代码中的 Gamma 校正表。 |

### 2.3 摄像头画面方向错误

**症状**：摄像头画面倒置 (180°) 或镜像。

| 解决方案 | 代码位置 |
|----------|----------|
| 画面上下颠倒 | 在固件中修改 `BSP_CAMERA_VFLIP` 宏：值 0/1 反转。 |
| 画面左右镜像 | 在固件中修改 `BSP_CAMERA_HMIRROR` 宏：值 0/1 反转。 |
| 两者都反 | 两个宏都调整。 |

> **不需要重新焊接**。这是软件配置问题。OV2640 的物理安装方向取决于排线弯折方式，不同批次可能不同。

## 3. 音频问题 | Audio Issues

### 3.1 完全无声

**症状**：固件运行正常，屏幕有显示，但喇叭没有任何声音。

| 排查步骤 | 操作 |
|----------|------|
| 1 | **检查 BOM 中 NC 标记的元件** — 如果误焊了设计图上标记 NC 的电阻/电容，可能导致音频通路被短路或断开。**拆除 NC 元件**。 |
| 2 | 检查 NS4150B 焊接 — 用放大镜检查 QFN 引脚是否有虚焊或连锡。 |
| 3 | 检查喇叭焊接 — 万用表蜂鸣档测试喇叭两端是否与 NS4150B 的 OUT+ / OUT- 导通。 |
| 4 | 万用表测喇叭电阻 — 断开电源后测量喇叭两端直流电阻，应约为 3-4Ω（4Ω 喇叭）。开路 = 喇叭坏或焊接断开。 |
| 5 | 检查 NS4150B 的 SD (Shutdown) 引脚 — 应为低电平 (GND)，如果被拉高则功放处于关断状态。 |
| 6 | 检查 ES8311 的 I2C 通信 — 用逻辑分析仪或 I2C 扫描确认 ES8311 是否响应 I2C 地址。 |
| 7 | 检查 I2S 引脚配置 — 确认 I2S BCLK/WS/DATA 引脚与代码中 `i2s_pin_config_t` 一致。 |

### 3.2 声音很小或失真

**症状**：有声音输出但音量很小，或者声音明显失真/杂音。

| 原因 | 解决方案 |
|------|----------|
| I2S 采样率不匹配 | 确认 ES8311 的配置采样率与 I2S 配置一致 (通常 16kHz 或 48kHz)。 |
| 功放增益不足 | 检查 NS4150B 增益设置电阻值。 |
| 喇叭腔体密封 | 3020 腔体喇叭需要密封安装，如果漏气会导致声音小、低频缺失。 |
| 供电纹波 | 在功放 VCC 脚对地加一个 100μF 电解电容滤波。 |

### 3.3 麦克风不收音

**症状**：语音识别完全无反应，录音无声。

| 排查步骤 | 操作 |
|----------|------|
| 1 | 检查 B4013AM422-42 咪头焊接 — 注意正负极（驻极体咪头有极性）。 |
| 2 | 检查 ES8311 焊接 — 尤其是 MICIN+ / MICIN- 引脚。 |
| 3 | 检查隔直电容 — 咪头与 ES8311 之间通常串联一个 1μF-10μF 的隔直电容，检查是否漏焊。 |
| 4 | 检查 I2C 配置 — ES8311 的 MIC 偏置电压和 PGA 增益是否配置正确。 |
| 5 | 用示波器探 MICIN 引脚 — 对着麦克风说话时，应看到小幅模拟信号波形。如果一直是平直电平，说明咪头或偏置电路有问题。 |

## 4. 电源问题 | Power Issues

### 4.1 Type-C 上电失败

**症状**：插入 USB-C 后完全无反应，电源指示灯不亮。

| 排查步骤 | 操作 |
|----------|------|
| 1 | **万用表蜂鸣档测 Type-C VBUS 到 GND 是否短路** — 这是最关键的一步。如果短路，说明 Type-C 焊接连锡或 PCB 电源层短路。 |
| 2 | 检查 Type-C CC1/CC2 引脚 — 各需通过 5.1kΩ 电阻下拉到 GND，否则 Type-C 充电器不会输出 5V。 |
| 3 | 用万用表测 Type-C 的 VBUS 引脚是否有 5V — 如果 CC 电阻正常但仍无 5V，可能是 Type-C 母座虚焊。 |
| 4 | 检查 DC-DC 芯片输入/输出 — 有输入 5V 但没有 3.3V 输出？DC-DC 芯片坏或焊接问题。 |

### 4.2 电池无法充电

**症状**：插着 USB-C，电池电量不增加或充电指示灯不亮。

| 排查步骤 | 操作 |
|----------|------|
| 1 | 万用表测充电 IC 的 VBUS_IN 脚是否有 5V。 |
| 2 | 万用表测充电 IC 的 BAT+ 输出电压 — 空载应有 ~4.2V（恒压充电电压）。 |
| 3 | 检查电池插头是否插紧 — 1.25mm 插头容易松脱。 |
| 4 | **万用表测电池电压** — 如果电池电压低于 2.5V，电池已过放损坏，需更换。 |
| 5 | 检查充电 IC 的 TS 引脚 — 如果使用 NTC 电阻进行温度检测，确保电阻值正确（通常常温 10kΩ）。如果 TS 直接接 GND，检查是否虚焊。 |
| 6 | 换一根 USB 线 — 有些劣质线压降大，到板上不到 4.5V，充电 IC 不启动。 |

### 4.3 电池很快耗尽

**症状**：2000mAh 电池只能用不到 30 分钟（正常应为 2-4 小时）。

| 原因 | 解决方案 |
|------|----------|
| PCB 有短路或微短路 | **断开电池**，用万用表测整板静态功耗。正常待机应 < 10mA。如果 > 100mA，逐模块断开排查。 |
| 某个模块一直高功耗 | 检查显示屏背光是否常亮、WiFi 是否持续发射、摄像头是否一直工作。固件中加入低功耗逻辑。 |
| 电池老化/虚标 | 换一块新电池测试。廉价聚合物锂电池实际容量可能远低于标称值。 |

## 5. 连接问题 | Connectivity Issues

### 5.1 WiFi 搜索不到 5GHz 网络

**症状**：只能搜索到 2.4GHz WiFi，看不到 5GHz WiFi。

**原因**：**ESP32-S3 的 WiFi 仅支持 2.4GHz 频段，不支持 5GHz**。这不是故障，是硬件限制。

**解决方案**：使用 2.4GHz WiFi 路由器，或开启路由器的 2.4GHz 频段（大多数路由器默认双频同时开启）。

### 5.2 百度 Token 失效

**症状**：AI 对话功能突然不可用，返回鉴权错误。

| 原因 | 解决方案 |
|------|----------|
| Token 已过期 | 百度 API Token 有效期 **30 天**，到期后需**重新获取**。到百度智能云控制台重新申请 Access Token。 |
| 免费额度用完 | 检查百度智能云账户的 API 调用量是否超过免费额度。 |

> 固件中通常不会自动刷新 Token。建议在云端部署一个简单的 Token 刷新服务，或定期手动更新。

### 5.3 蓝牙音箱无法配网

**症状**：想通过蓝牙连接外部音箱，但配网失败。

**原因**：蓝牙音箱（如小度音箱）有自己独立的 WiFi 配网通道，不通过 ESP-SparkBot 配网。

**解决方案**：使用音箱厂商的 APP（如小度 APP）对音箱重新配网，确保音箱和 ESP-SparkBot 在**同一 2.4GHz WiFi 网络**下。

### 5.4 履带底盘无响应

**症状**：底盘不运动，或偶尔响应偶尔不响应。

| 排查步骤 | 操作 |
|----------|------|
| 1 | 检查磁吸接口是否吸合牢固 — 4 个触点是否全部接触。 |
| 2 | 万用表测磁吸接口 Pin 1 (5V) 和 Pin 4 (GND) — 确认底盘已供电。 |
| 3 | 检查 UART 通信 — 用逻辑分析仪或示波器测 Pin 2 (TX) 和 Pin 3 (RX) 是否有信号。 |
| 4 | 确认波特率匹配 — 核心板和底盘 ESP8684 的 UART 波特率必须一致。 |
| 5 | 底盘独立供电测试 — 将底盘从机器人上取下，单独给底盘供电，用 USB-TTL 工具直接测试 ESP8684 串口通信。 |

## 6. 触摸问题 | Touch Issues

### 6.1 触摸按键不灵敏

**症状**：需要反复触摸才有反应，或完全无反应。

| 排查步骤 | 操作 |
|----------|------|
| 1 | **执行基线校准** — ESP32-S3 触摸传感器上电时需要读取无触摸时的基线值。如果上电时手指正好放在触摸区域，基线会偏移导致检测失败。断开电源，确保手指离开，重新上电。 |
| 2 | 调整滤波参数 — 在固件中降低触摸滤波的平滑系数（减少平均窗口长度），提高响应速度。 |
| 3 | 调整触摸阈值 — 增大 `touch_pad_set_thresh()` 的灵敏度阈值。默认值可能太保守。 |
| 4 | 检查 FPC 排线焊接 — 触摸 FPC 排线虚焊会导致电容值漂移。 |
| 5 | 检查外壳厚度 — 如果 3D 打印外壳触摸区域太厚（>2mm），透过外壳的触摸电容变化太弱。打磨减薄或改用更灵敏的参数。 |

## 7. 装配问题 | Assembly Issues

### 7.1 核心板与底板插反烧板

**症状**：插上后上电瞬间冒烟、芯片发烫、闻到焦味。

**原因**：核心板与底板的 1.27mm 排针排母没有方向防呆设计，插反会导致 VCC 和 GND 交换，直接烧毁板上的芯片。

**预防措施**：
- 插接前**务必对照 PCB 丝印上的方向标记**。
- 插好后**先用万用表蜂鸣档测 VCC-GND 是否短路**，确认无误再上电。
- 如果已经在板子上做了标记（如用记号笔在排针一侧画线），每次插拔前对齐标记。

**如果已经烧了**：
- 立即断电。
- 核心板大概率 ESP32-S3 模组已烧毁（最贵的一个件），需要更换。
- 底板上的 DC-DC 和充电 IC 也容易烧毁。
- 建议重新打板 + 重新焊接（教训代价 > 100 元）。

### 7.2 排针弯曲/无法插入

**症状**：核心板无法顺利插入底板排母，排针歪斜。

| 原因 | 解决方案 |
|------|----------|
| 焊接时排针未对齐 | 用镊子小心掰直排针（动作要轻，避免折断或焊盘脱落）。如果弯曲严重，拆焊重新焊排针。 |
| 板厚不对 | 排针排母选型与 1.0mm 板厚不匹配。确认排针排母为 1.27mm 间距规格。 |

### 7.3 屏幕排线折断

**症状**：屏幕完全不显示或不完全显示（部分行/列缺失）。

**原因**：ST7789 屏幕的 FPC 排线非常脆弱，反复弯折或装配时用力不当会导致内部铜线断裂。

**预防**：装配时先插好排线再固定屏幕位置，尽量减少排线弯折次数。
**解决**：排线断裂无法修复，换新屏幕。

## 8. 快速诊断流程图 | Quick Diagnostic Flowchart

```
上电无反应？
├─ 是 → 测 USB-C VBUS 有 5V？
│       ├─ 无 → Type-C 焊接问题 / USB线问题
│       └─ 有 → 测 3.3V 输出？
│               ├─ 无 → DC-DC 问题
│               └─ 有 → ESP32-S3 焊接 / 模组问题
│
└─ 否 (有反应但...) →
    屏幕不显示？
    ├─ 背光亮? → 是 → SPI焊接/配置问题
    └─ 无背光 → 供电/屏幕问题
    
    无声音？
    ├─ I2C能扫到 ES8311? → 否 → I2C焊接问题
    ├─ 喇叭两端电阻 ~3-4Ω? → 否 → 喇叭断线
    └─ 有电阻 → NS4150B 焊接/配置问题
    
    WiFi 连不上？
    ├─ 搜不到任何网络? → 模组天线问题
    └─ 只能搜到 2.4G? → 正常 (ESP32-S3 不支持 5GHz)
```

## 调试工具速查 | Debug Tools Quick Reference

| 工具 | 用途 | 使用场景 |
|------|------|----------|
| 万用表 (蜂鸣档) | 通断测试、短路检测 | **每次上电前** |
| 万用表 (电压档) | 测各电压节点 | 电源故障排查 |
| 万用表 (电阻档) | 测对地阻抗 | 判断是否短路 |
| USB-TTL 工具 | 独立串口调试 | 怀疑 USB-UART 有问题时 |
| 逻辑分析仪 | 抓取 I2C/SPI/I2S 波形 | 通信异常分析 |
| 示波器 | 观察模拟信号波形 | 音频调试、电源纹波 |
| 放大镜/显微镜 | 检查密脚焊接 | QFN 芯片、FPC 座 |
| I2C 扫描脚本 | 检测 I2C 总线上设备 | BMI270、ES8311 不响应时 |

**I2C 扫描代码 (Arduino/ESP-IDF 通用思路)**：
```c
// 扫描 I2C 总线上的所有设备
// 预期结果：
//   0x18 - ES8311
//   0x68 或 0x69 - BMI270
//   0x3C 或 0x3D - 某些 OLED/屏幕 (如果使用 I2C 转接)
for (uint8_t addr = 1; addr < 127; addr++) {
    i2c_master_write_to_device(I2C_NUM_0, addr, NULL, 0, 100 / portTICK_PERIOD_MS);
    if (err == ESP_OK) {
        ESP_LOGI(TAG, "I2C device found at 0x%02X", addr);
    }
}
```
