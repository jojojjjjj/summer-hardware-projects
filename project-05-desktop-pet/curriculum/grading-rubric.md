# 评分标准 | Grading Rubric

## 总体评分体系 | Overall Grading System

| 维度 | 权重 | 说明 |
|------|------|------|
| 技术实现 | 40% | 功能完整度、代码质量、bug修复能力 |
| 文档质量 | 15% | README、代码注释、设计文档 |
| 演示展示 | 15% | Final Demo清晰度、表达能力、Q&A应对 |
| Demo网站 | 15% | 项目展示网站的可访问性、内容完整性、视觉呈现 |
| 进度汇报 | 10% | 每周Check-in质量、问题分析深度 |
| 创新拓展 | 10% | 额外功能、创意设计、个性化定制 |

---

## 技术实现（40%）| Technical Implementation

### 评分细则

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | 所有核心功能正常运行，代码结构清晰，有创新性功能，无明显bug |
| 良好 | 80-89 | 核心功能基本完整，代码可读性好，有少量不影响体验的bug |
| 合格 | 70-79 | 主要功能实现，但缺少1-2个核心功能，或有影响体验的bug |
| 需改进 | 60-69 | 仅完成基础窗口和动画，缺少状态机或交互功能 |
| 不合格 | <60 | 代码无法正常运行，或仅完成框架 |

### 功能检查清单

| 功能 | 分值 | 说明 |
|------|------|------|
| 透明无边框窗口 | 5分 | 桌宠在透明窗口中正常显示 |
| 帧动画播放 | 5分 | 至少2种状态的动画（如idle和walk） |
| 鼠标交互 | 5分 | 点击和拖拽响应 |
| 状态机 | 8分 | 至少3个状态的状态机实现 |
| 宠物自主行为 | 7分 | 宠物能自主移动和切换行为 |
| 对话气泡 | 5分 | 气泡框显示文字 |
| 音效系统（加分） | 3分 | 添加了音效反馈 |
| AI对话（加分） | 2分 | 集成了LLM对话功能 |

---

## 文档质量（15%）| Documentation Quality

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | README完整、注释充分、有架构说明和使用示例 |
| 良好 | 80-89 | README基本完整、关键代码有注释 |
| 合格 | 70-79 | 有README但内容简单、部分代码有注释 |
| 需改进 | 60-69 | README过于简单、几乎没有代码注释 |
| 不合格 | <60 | 没有README、没有注释 |

### 文档检查清单

| 项目 | 分值 | 说明 |
|------|------|------|
| README.md | 5分 | 包含项目简介、使用说明、截图 |
| 代码注释 | 5分 | 关键函数和复杂逻辑有中文注释 |
| 设计说明 | 5分 | 能够解释状态机设计、动画系统架构 |
| Git提交记录 | 5分 | 有意义的commit message，合理的提交频率 |

---

## 演示展示（15%）| Final Presentation

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | 演示流畅完整，表达清晰，能深入回答技术问题 |
| 良好 | 80-89 | 演示基本完整，表达较清楚，能回答基本问题 |
| 合格 | 70-79 | 演示了主要功能，但讲解不够清晰 |
| 需改进 | 60-69 | 演示有明显遗漏或多次出错 |
| 不合格 | <60 | 未能完成演示 |

### 演示评分项

| 项目 | 分值 | 说明 |
|------|------|------|
| 功能演示 | 8分 | 流畅地展示桌宠各项功能 |
| 技术讲解 | 4分 | 能够解释关键技术和设计决策 |
| 问题回答 | 4分 | 能够回答老师和同学的提问 |
| 时间控制 | 2分 | 在规定时间内完成演示 |
| 创意展示 | 2分 | 展示个性化功能或创意设计 |

---

## Demo网站（15%）| Project Demo Website

| 评价项 | 分值 | 说明 |
|--------|------|------|
| 网站可正常访问（无404错误） | 15 | 部署成功，URL有效 |
| 包含项目介绍（名称、目标、核心功能） | 10 | 清晰描述项目背景 |
| 包含桌宠展示（角色设计、动画截图/GIF） | 10 | 有视觉化的项目展示 |
| 包含软件架构说明（系统框图、状态机设计） | 10 | 展示技术理解深度 |
| 包含演示视频或GIF | 15 | 功能演示直观 |
| 包含技术亮点说明 | 10 | 突出个人创新 |
| 包含学习心得与总结 | 10 | 体现学习收获 |
| 包含B站参考视频链接列表 | 5 | 参考资源整理 |
| 页面整洁、排版合理、图文并茂 | 10 | 视觉呈现质量 |
| 对核心技术的解释清晰准确 | 5 | 技术表达能力 |
| **总分** | **/100** | |

## 进度汇报（10%）| Progress Reports

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | 内容详实，有深入的反思和下一步计划 |
| 良好 | 80-89 | 内容完整，描述了进展和遇到的问题 |
| 合格 | 70-79 | 基本描述了进展，但缺少深度分析 |
| 需改进 | <70 | 内容过于简单或未按时提交 |

## 创新拓展（10%）| Innovation & Extensions

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | 有2个以上创新功能，且实现质量高 |
| 良好 | 80-89 | 有1个创新功能，实现较完整 |
| 合格 | 70-79 | 有个性化定制（如自定义角色、新的动画） |
| 需改进 | <70 | 没有任何个性化内容 |

### 加分项参考

| 创新功能 | 额外加分 | 说明 |
|----------|----------|------|
| AI智能对话 | +5分 | 集成LLM API实现自然语言对话 |
| 硬件互动 | +5分 | 通过Arduino传感器与桌宠互动 |
| 多宠物模式 | +3分 | 支持同时显示多个宠物 |
| 自定义角色编辑器 | +3分 | 能够导入自定义角色素材 |
| 天气响应 | +2分 | 根据真实天气改变宠物行为 |
| 桌面小工具 | +2分 | 集成时钟、天气等桌面小功能 |
