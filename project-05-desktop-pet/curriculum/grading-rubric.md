# 评分标准 | Grading Rubric

## 总体评分体系 | Overall Grading System

| 维度 | 权重 | 说明 |
|------|------|------|
| 技术实现 | 40% | 硬件组装质量、固件功能完整度、代码质量 |
| 文档质量 | 15% | README、代码注释、接线图/原理图标注 |
| 演示展示 | 15% | Final Demo清晰度、表达能力、Q&A应对 |
| Demo网站 | 15% | 项目展示网站的可访问性、内容完整性、视觉呈现 |
| 进度汇报 | 10% | 每周Check-in质量、问题分析深度 |
| 创新拓展 | 10% | 额外功能、创意设计、个性化定制 |

---

## 技术实现（40%）| Technical Implementation

### 评分细则

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | 硬件焊接质量高，所有核心功能正常运行，代码结构清晰，有创新性功能，无明显bug |
| 良好 | 80-89 | 硬件组装完整，核心功能基本正常，代码可读性好，有少量不影响体验的bug |
| 合格 | 70-79 | 主要功能实现，但缺少1-2个核心功能，或有影响体验的bug |
| 需改进 | 60-69 | 仅完成基础硬件组装和简单固件，缺少多个核心功能 |
| 不合格 | <60 | 硬件无法正常工作，或固件无法编译运行 |

### 功能检查清单

| 类别 | 功能 | 分值 | 说明 |
|------|------|------|------|
| 硬件 | PCB焊接质量 | 5分 | 焊点光滑、无虚焊、无短路 |
| 硬件 | 整机组装完整 | 3分 | 3块PCB正确连接，外壳安装牢固 |
| 硬件 | 上电测试通过 | 2分 | 各电压测试点正常，可烧录固件 |
| 固件 | GPIO/LED控制 | 3分 | LED灯、触摸按键正常 |
| 固件 | BMI270传感器数据读取 | 3分 | I2C通信正常，能读取加速度/陀螺仪数据 |
| 固件 | ST7789显示屏驱动 | 5分 | 屏幕显示正常，LVGL界面可交互 |
| 固件 | OV2640摄像头驱动 | 3分 | 能够拍照并在屏幕上显示 |
| 固件 | ES8311音频系统 | 5分 | 录音和播放功能正常 |
| 固件 | WiFi联网功能 | 3分 | 能够连接WiFi，NTP时间同步 |
| 固件 | 天气时钟功能 | 3分 | 获取并显示天气信息 |
| 固件 | AI对话集成 | 5分 | LLM API调用、TTS输出正常 |
| 固件 | 蓝牙音箱功能 | 2分 | A2DP蓝牙音频播放 |
| 固件 | 语音唤醒 | 3分 | 唤醒词检测响应正常 |

## 文档质量（15%）| Documentation Quality

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | README完整、代码注释充分、有架构图和接线图 |
| 良好 | 80-89 | README基本完整、关键代码有注释 |
| 合格 | 70-79 | 有README但内容简单、部分代码有注释 |
| 需改进 | 60-69 | README过于简单、几乎没有代码注释 |
| 不合格 | <60 | 没有README、没有注释 |

### 文档检查清单

| 项目 | 分值 | 说明 |
|------|------|------|
| README.md | 5分 | 包含项目简介、硬件配置、编译烧录说明、功能截图 |
| 代码注释 | 5分 | 关键函数和复杂逻辑有中文注释 |
| 接线图/框图 | 5分 | 有系统框图或模块连接说明 |
| Git提交记录 | 5分 | 有意义的commit message，合理的提交频率 |

## 演示展示（15%）| Final Presentation

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | 演示流畅完整，硬件和软件都运行正常，能深入回答技术问题 |
| 良好 | 80-89 | 演示基本完整，表达较清楚，能回答基本问题 |
| 合格 | 70-79 | 演示了主要功能，但讲解不够清晰或部分功能未展示 |
| 需改进 | 60-69 | 演示有明显遗漏或多次故障 |
| 不合格 | <60 | 未能完成演示 |

### 演示评分项

| 项目 | 分值 | 说明 |
|------|------|------|
| 功能演示 | 8分 | 流畅地展示SparkBot各项核心功能 |
| 技术讲解 | 4分 | 能够解释关键技术和设计决策 |
| 问题回答 | 4分 | 能够回答老师和同学的提问 |
| 时间控制 | 2分 | 在规定时间内完成演示 |
| 创意展示 | 2分 | 展示个性化功能或创意设计 |

## Demo网站（15%）| Project Demo Website

| 评价项 | 分值 | 说明 |
|--------|------|------|
| 网站可正常访问（无404错误） | 15 | 部署成功，URL有效 |
| 包含项目介绍（名称、目标、核心功能） | 10 | 清晰描述项目背景 |
| 包含硬件展示（PCB照片、组装过程照片） | 10 | 有视觉化的硬件展示 |
| 包含系统架构说明（框图、模块说明） | 10 | 展示技术理解深度 |
| 包含演示视频或GIF | 15 | 功能演示直观 |
| 包含技术亮点说明 | 10 | 突出个人创新 |
| 包含学习心得与总结 | 10 | 体现学习收获 |
| 包含参考资源链接 | 5 | B站视频、文档链接等 |
| 页面整洁、排版合理、图文并茂 | 10 | 视觉呈现质量 |
| 对核心技术的解释清晰准确 | 5 | 技术表达能力 |
| **总分** | **/100** | |

## 进度汇报（10%）| Progress Reports

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | 内容详实，有深入的问题反思和下一步计划 |
| 良好 | 80-89 | 内容完整，描述了进展和遇到的问题 |
| 合格 | 70-79 | 基本描述了进展，但缺少深度分析 |
| 需改进 | <70 | 内容过于简单或未按时提交 |

### 汇报评分项

| 项目 | 分值 | 说明 |
|------|------|------|
| 进度描述 | 5分 | 清晰描述本周硬件和软件完成情况 |
| 问题分析 | 3分 | 对遇到的问题有深入分析 |
| 下周计划 | 2分 | 有明确的下一步计划 |

## 创新拓展（10%）| Innovation & Extensions

| 等级 | 分数范围 | 标准 |
|------|----------|------|
| 优秀 | 90-100 | 有2个以上创新功能，且实现质量高 |
| 良好 | 80-89 | 有1个创新功能，实现较完整 |
| 合格 | 70-79 | 有个性化定制（如自定义LVGL界面、自定义外壳） |
| 需改进 | <70 | 没有任何个性化内容 |

### 加分项参考

| 创新功能 | 额外加分 | 说明 |
|----------|----------|------|
| 自定义AI角色 | +5分 | 设计独特的AI角色和对话风格 |
| 自定义LVGL界面 | +5分 | 设计独特的UI主题和动画 |
| 小游戏开发 | +5分 | 如木鱼、2048、骰子等交互游戏 |
| ESP-NOW多机同步 | +3分 | 两台SparkBot通过ESP-NOW通信 |
| 自定义3D外壳 | +3分 | 自行设计或修改3D打印外壳 |
| USB投屏副屏 | +3分 | 实现PC副屏显示功能 |
| 履带小车扩展 | +5分 | 接入电机驱动，实现移动功能 |
