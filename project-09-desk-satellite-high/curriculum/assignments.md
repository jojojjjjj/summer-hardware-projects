# 作业说明 | Assignments Overview

## HW1：第一周进度汇报 | Week 1 Check-in (Day 6)

### 要求 Requirements

- **字数**：800-1200 字（中文或英文）| 800-1200 words (Chinese or English)
- **提交格式**：Markdown 文件，提交到 `assignments/week-1-checkin.md`

### 必须包含的内容 Required Content

1. **已完成的功能列表**（配截图/照片）
   - 列出已成功实现的功能
   - 每个功能附照片或串口截图
   - 标注完成日期

2. **遇到的主要困难和解决方法**
   - 至少描述 3 个技术问题
   - 每个问题说明：现象 → 原因分析 → 解决方法
   - 记录调试过程中使用的工具和方法

3. **第二周计划**
   - 列出 Day 7-12 每天的预期目标
   - 标注关键里程碑
   - 识别可能的风险点

4. **个人反思**
   - 学到了什么新知识/技能
   - 哪里可以做得更好
   - 对项目的感受和期待

### 评分重点 Grading Focus

- 内容的详细程度和真实性（40%）
- 问题分析的深度（30%）
- 计划的可行性（20%）
- 反思的真诚度（10%）

## HW3：最终演示 | Final Demo (Day 12)

### 要求 Requirements

- **时长**：5-8 分钟口头演示 + 3 分钟 Q&A
- **语言**：中文或英文均可，鼓励使用中英混合
- **形式**：实物展示 + 口头讲解（可辅助 PPT/网页）

### 必须包含的环节 Required Segments

1. **项目背景和目标**（1 分钟）
   - 为什么选择这个项目
   - 项目灵感来源（Huy Vector）
   - 期望实现的功能

2. **技术架构概述**（1 分钟）
   - 硬件选型理由
   - 软件架构图
   - 通信协议（SPI + I2C）

3. **硬件实物展示 + 功能演示**（2-3 分钟）
   - 展示卫星外壳实物
   - 现场演示：时间同步、温度/湿度显示、充电功能
   - 展示太阳能充电效果（如有条件）

4. **关键技术点讲解**（1-2 分钟）
   - 选择 1-2 个技术难点深入讲解
   - 例如：SPI 显示驱动、电源管理、铜线弯制技巧

5. **遇到的问题与解决方案**（1 分钟）
   - 最具挑战性的问题
   - 创造性的解决方案
   - 学到的教训

### 评分标准 Grading Criteria

| 维度 Dimension | 权重 Weight | 说明 Description |
|---------------|-----------|-----------------|
| 演示清晰度 Clarity | 40% | 逻辑清晰、语言流畅、节奏得当 |
| 功能完整度 Completeness | 30% | 实物功能正常、演示顺利 |
| Q&A 应对能力 Q&A Response | 20% | 回答准确、能承认不足 |
| 时间控制 Time Management | 10% | 在规定时间内完成 |

## HW4：Demo 网站 | Project Demo Website (Day 12)

### 要求 Requirements

- **形式**：独立网页，可使用任意静态网站生成器
- **部署**：必须部署到公网可访问的地址（GitHub Pages / Vercel / Cloudflare Pages / Netlify 均可）

### 必须包含的板块 Required Sections

1. **项目介绍页**
   - 项目名称、简介、灵感来源
   - 最终效果照片（多角度、点亮状态）
   - 项目规格参数表

2. **功能展示**
   - 每个核心功能的展示（图片/视频/GIF）
   - 时钟同步、温度/湿度、太阳能充电等

3. **技术架构说明**
   - 硬件架构图（ESP32-C3 + ST7735 + SHT31-D + 电源系统）
   - 软件流程图
   - 通信协议说明（SPI + I2C）

4. **制作过程和心得**
   - Day-by-day 简要记录
   - 关键照片（焊接过程、组装过程）
   - 遇到的挑战和解决方案
   - 个人/团队感想

5. **团队信息**（如适用）
   - 成员介绍
   - 分工说明

### 技术要求 Technical Requirements

- 响应式设计（手机和电脑均可浏览）
- 所有图片使用压缩后的版本（每张 < 500KB）
- 至少 3 个页面或 3 个独立板块
- 包含项目的 GitHub 仓库链接

### 推荐工具 Recommended Tools

- **GitHub Pages** + Markdown -- 最简单
- **Vercel** + Next.js -- 进阶选择
- **Cloudflare Pages** + Astro -- 性能优秀
- **纯 HTML/CSS/JS** -- 自由度最高

### 评分标准 Grading Criteria

| 维度 Dimension | 权重 Weight | 说明 Description |
|---------------|-----------|-----------------|
| 内容完整性 Content | 40% | 所有板块齐全、信息准确 |
| 视觉呈现 Visual | 30% | 布局美观、图片质量好 |
| 技术深度 Technical Depth | 20% | 架构图清晰、技术说明准确 |
| 部署状态 Deployment | 10% | 网站可正常访问、响应式正常 |

## 提交规范 | Submission Guidelines

### Git 提交规范 Git Commit Convention

```
feat: 新功能 (new feature)
fix: 修复 bug (bug fix)
docs: 文档更新 (documentation)
style: 代码格式 (formatting)
refactor: 重构 (code restructuring)
test: 测试 (testing)
chore: 构建/工具 (build/tools)
```

### 文件命名 File Naming

- 进度汇报：`assignments/week-1-checkin.md`, `assignments/week-2-checkin.md`
- 代码：`software/src/desk_satellite.ino`
- 配置：`software/config.template.h`
- 文档：`README.md`, `hardware/wiring-guide.md`
