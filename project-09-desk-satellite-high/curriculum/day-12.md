**Q&A 准备**：

```
常见问题及参考回答：

Q: 为什么选择 ESP32-C3 而不是 ESP8266？
A: ESP32-C3 使用 RISC-V 架构，性能更强（160MHz），
   内置 BLE 5.0，Flash 更大（4MB），GPIO 更多。
   虽然价格稍高，但扩展性更好。

Q: SPI 和 I2C 的区别是什么？
A: SPI 使用 4 线（SCLK, MOSI, MISO, CS），速度快，适合显示屏。
   I2C 使用 2 线（SCL, SDA），接线简单，适合传感器。
   我们的项目同时使用两种协议。

Q: 铜线外壳最难的部分是什么？
A: 保持框架的对称性和焊点的美观。
   铜线弯制需要耐心和力度控制。

Q: 电池续航多久？
A: 连续工作约 6-8 小时。使用 Deep Sleep 可以延长到数天。
   加上太阳能板在晴天可以持续供电。

Q: 温度精度如何保证？
A: SHT31-D 精度为 ±0.3°C。
   我们还做了简单的校准和滑动平均滤波来减少波动。
   注意将传感器远离 ESP32 芯片以避免热干扰。

Q: 如果重新做，你会改变什么？
A: （根据实际情况回答，诚实反思）
```

**部署步骤（以 GitHub Pages 为例）**：

```
GitHub Pages 部署：

1. 创建 GitHub 仓库
   git init
   git add .
   git commit -m "Desk Satellite project"

2. 推送到 GitHub
   git remote add origin https://github.com/YOUR_USERNAME/desk-satellite.git
   git push -u origin main

3. 启用 GitHub Pages
   - 进入仓库 Settings -> Pages
   - Source: Deploy from a branch
   - Branch: main, / (root)
   - Save

4. 等待部署完成（约 1-2 分钟）
   - 访问 https://YOUR_USERNAME.github.io/desk-satellite/

5. 验证网站
   - 检查所有页面是否正常显示
   - 检查图片是否加载
```
