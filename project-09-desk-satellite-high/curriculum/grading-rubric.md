# 详细评分标准 | Detailed Grading Rubric

## 评分维度 | Grading Dimensions

本项目总分由 6 个维度加权计算，满分为 100 分。

> The total score is calculated from 6 weighted dimensions, with a maximum of 100 points.

---

### 1. 技术实现 (35%) | Technical Implementation

| 等级 Grade | 分数 Score | 标准 Criteria |
|-----------|-----------|--------------|
| **A - 优秀 Excellent** | 90-100 | 所有核心功能正常：NTP 时钟同步精确、SHT31-D 温湿度数据准确、TFT 彩色显示稳定、WiFi 连接可靠、电源管理（锂电池 + 太阳能）工作正常、触摸开关控制正常。代码结构清晰，有拓展功能（如动画效果、多屏切换、电池电量显示），无明显 bug。卫星外壳结构美观坚固。 |
| **B - 良好 Good** | 75-89 | 核心功能完整（时钟 + 温湿度 + 显示 + WiFi），偶有小 bug 但已修复。代码结构基本清晰。卫星外壳已组装完成。太阳能充电或电量指示有一项未完成不影响主要功能。 |
| **C - 合格 Pass** | 60-74 | 基本功能可用（至少时钟 + 温湿度能显示在 TFT 上），存在已知 bug 但已记录在文档中。WiFi 连接偶尔不稳定。卫星外壳基本成型但不够精致。 |
| **D - 待改进 Needs Improvement** | < 60 | 核心功能未完成或无法运行。TFT 无显示、传感器无数据、WiFi 无法连接等关键问题未解决。 |

**具体考核点 Specific Assessment Points**：

| 功能模块 Module | 考核内容 Assessment Content | 权重 Weight |
|----------------|--------------------------|------------|
| TFT 显示 TFT Display | 能正确显示彩色文字、图形、时间、温湿度数据 | 25% |
| SHT31-D 传感器 Sensor | 温度精度 ±1°C 以内，湿度精度 ±5% 以内 | 15% |
| WiFi + NTP WiFi & NTP | 能连接 WiFi 并同步网络时间，误差 < 2 秒 | 20% |
| 电源管理 Power | 锂电池充电正常，Type-C 充电可用 | 15% |
| 交互控制 Interaction | 触摸开关控制正常，LED 指示灯工作 | 10% |
| 代码质量 Code Quality | 结构清晰、注释充分、无明显浪费 | 15% |

---

### 2. 演示展示 (17%) | Presentation

| 等级 Grade | 分数 Score | 标准 Criteria |
|-----------|-----------|--------------|
| **A - 优秀** | 90-100 | 演示流畅，卫星实物正常运转。技术讲解深入，能清楚解释 SPI/I2C 通信原理和电源管理系统。Q&A 对答如流，能回答深入的技术问题。时间控制精准（5-8 分钟内完成）。 |
| **B - 良好** | 75-89 | 演示基本顺畅，实物能正常工作。技术讲解清晰，能说明核心功能的实现方式。能回答大部分问题。时间控制基本得当。 |
| **C - 合格** | 60-74 | 演示有卡顿，实物部分功能异常（但能解释原因）。讲解不够清晰或过于简略。对 Q&A 中较难的问题无法回答。 |
| **D - 待改进** | < 60 | 无法完成演示，或实物完全不能工作。讲解混乱，无法回答基本问题。严重超时或过短。 |

---

### 3. 进度汇报 (17%) | Check-ins

| 等级 Grade | 分数 Score | 标准 Criteria |
|-----------|-----------|--------------|
| **A - 优秀** | 90-100 | 两次汇报均按时提交，内容详实（1200 字左右）。问题分析深入（至少 3 个问题的现象-原因-解决方案）。反思有深度，计划清晰可行。附有充分的截图和照片。 |
| **B - 良好** | 75-89 | 两次汇报按时提交，内容完整（800 字以上）。有基本的问题分析和反思。有部分截图。计划基本合理。 |
| **C - 合格** | 60-74 | 仅提交一次汇报，或两次都较敷衍（< 800 字）。问题分析浅显，缺乏具体细节。 |
| **D - 待改进** | < 60 | 未提交，或提交的内容明显敷衍（< 300 字）。 |

---

### 4. Demo 网站 (13%) | Project Demo Website

| 等级 Grade | 分数 Score | 标准 Criteria |
|-----------|-----------|--------------|
| **A - 优秀** | 90-100 | 网站内容完整（介绍、功能展示、技术架构、制作过程、心得）。视觉设计美观，响应式布局完美。已部署可访问。包含硬件架构图和软件流程图。照片质量好（多角度、点亮/熄灭状态）。 |
| **B - 良好** | 75-89 | 内容基本完整，已部署可访问。视觉设计整洁。有基本的架构说明和制作过程记录。 |
| **C - 合格** | 60-74 | 内容不完整（缺少 1-2 个板块），或未成功部署。视觉设计简陋但可浏览。 |
| **D - 待改进** | < 60 | 无 Demo 网站，或内容严重缺失。无法访问。 |

---

### 5. 文档质量 (9%) | Documentation

| 等级 Grade | 分数 Score | 标准 Criteria |
|-----------|-----------|--------------|
| **A - 优秀** | 90-100 | README 完整详细。代码有清晰的中文/英文注释（每个函数、关键逻辑块）。接线图/原理图规范清晰。有调试指南和常见问题排查表。包含完整的项目结构说明。 |
| **B - 良好** | 75-89 | README 完整。代码有一定注释。接线图基本清晰。有基本的调试说明。 |
| **C - 合格** | 60-74 | README 存在但内容不完整。代码注释较少。接线图模糊或缺失。 |
| **D - 待改进** | < 60 | 几乎没有文档，代码无注释。 |

---

### 6. 团队协作 (9%) | Collaboration

| 等级 Grade | 分数 Score | 标准 Criteria |
|-----------|-----------|--------------|
| **A - 优秀** | 90-100 | Git 提交记录规律（几乎每天有提交），commit message 规范。分工明确（如一人电路、一人结构），配合默契。有互助记录（帮助其他组解决问题）。积极沟通。 |
| **B - 良好** | 75-89 | Git 有提交记录。分工基本合理。有基本沟通。 |
| **C - 合格** | 60-74 | Git 提交较少或不规律。分工不够明确。沟通不足。 |
| **D - 待改进** | < 60 | 无 Git 记录。分工混乱，一人完成几乎所有工作。 |

> **注**：独立完成项目的学员，此 9% 转换为"自主学习能力"评分，考察独立解决问题、自主查阅资料和自我管理的能力。
> **Note**: For independent students, this 9% converts to "Self-directed Learning", assessing independent problem-solving, self-directed research, and self-management.

---

## 总分计算 | Total Score Calculation

```
总分 Total Score =
    技术实现 Technical     x 0.35
  + 演示展示 Presentation  x 0.17
  + 进度汇报 Check-ins     x 0.17
  + Demo 网站 Website      x 0.13
  + 文档质量 Documentation x 0.09
  + 团队协作 Collaboration  x 0.09
```

### 等级换算 | Grade Conversion

| 总分范围 Score Range | 等级 Grade | 说明 Description |
|---------------------|-----------|-----------------|
| 90-100 | A（优秀 Excellent） | 全方位出色 |
| 80-89 | B+（良好+ Very Good） | 多数维度优秀 |
| 75-79 | B（良好 Good） | 整体表现良好 |
| 70-74 | B-（良好- Above Average） | 部分维度需加强 |
| 60-69 | C（合格 Pass） | 基本达标但有改进空间 |
| < 60 | D（待改进 Needs Improvement） | 需要补做或重修 |

## 加分项 | Bonus Points

以下情况可获得额外加分（最多 +10 分，总分不超过 100 分）：

| 加分项 Bonus Item | 加分 Points | 说明 |
|------------------|------------|------|
| 太阳能充电实际可用 Solar charging works | +3 | 在日光下能稳定为电池充电 |
| 创意外观设计 Creative enclosure design | +2 | 铜线外壳设计独特、超出基本要求 |
| 帮助其他组 Helped other teams | +2 | 有记录的互助行为 |
| 开源贡献 Open source contribution | +3 | 提交 PR 到 Adafruit 库或发布自己的教程 |
| Demo 网站使用英文 English Demo Website | +1 | 网站内容使用英文撰写 |
