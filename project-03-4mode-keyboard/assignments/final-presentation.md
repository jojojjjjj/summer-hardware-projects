## 🌐 项目展示网站 | Project Demo Website

**这是最终评分的重要组成部分。** 你必须创建一个在线可访问的项目展示网站。

**This is a critical part of your final grade.** You must create an online-accessible project demo website.

### 要求 | Requirements

1. **部署平台** — 使用以下任一免费平台部署：
   - GitHub Pages（推荐）— 推送 HTML 到仓库的 `docs/` 分支或 `gh-pages` 分支
   - Vercel — 连接 GitHub 仓库自动部署
   - Netlify — 拖拽文件夹即可部署
   - 其他可公网访问的平台均可

2. **网站内容必须包含：**
   - [ ] 项目介绍（名称、目标、核心功能）
   - [ ] 硬件展示（BOM清单、实物照片/渲染图、接线图）
   - [ ] 软件架构（系统框图、模块说明、关键代码片段）
   - [ ] 演示视频或 GIF（展示键盘实际运行效果）
   - [ ] 12天学习心得与总结
   - [ ] B站参考视频链接列表

3. **技术建议：**
   - 最简方案：一个 `index.html` 文件 + CSS 样式即可
   - 可使用 GitHub Pages 的 Jekyll 主题快速生成
   - 推荐工具：Markdown + 静态网站生成器（如 VitePress、Hugo）

### 提交方式 | Submission

- 在最终展示前，将网站 URL 提交给老师
- URL 格式示例：`https://你的用户名.github.io/project-name/`
- 老师将通过访问该 URL 进行评分

### 评分标准 | Scoring

| 维度 | 分值 | 说明 |
|------|------|------|
| 网站可访问性 | 5分 | URL可正常打开，无404错误 |
| 内容完整性 | 5分 | 包含所有要求的内容模块 |
| 视觉呈现 | 3分 | 页面整洁、排版合理、图文并茂 |
| 技术展示深度 | 2分 | 对核心技术的解释清晰准确 |
| **小计** | **15分** | |

## 展示大纲 | Presentation Outline

### 第一部分：开场 (1 分钟)

**目标**：吸引注意力，建立项目背景

- 自我介绍（姓名、班级）
- 项目名称和一句话概述
- 选题动机（为什么选择做宏键盘？）

**PPT 建议**：1 页，包含项目名称和最终成品照片

---

### 第二部分：项目概述 (2 分钟)

**目标**：让听众快速了解你做了什么

- 最终效果展示（照片或短视频）
- 技术栈概述（硬件 + 软件）
- 项目时间线回顾

**PPT 建议**：2-3 页
- 一页：功能列表（16 键、旋钮、OLED、RGB）
- 一页：技术栈概览图
- 一页：12 天时间线

---

### 第三部分：技术讲解 (5 分钟)

**目标**：展示你对核心技术的理解深度

**必讲内容 Required Topics:**

1. **矩阵扫描原理**（1 分钟）
   - 为什么用矩阵而不是直连？
   - 二极管防鬼键的原理
   - 扫描过程的动画或示意图

2. **PCB 设计**（1.5 分钟）
   - 原理图的关键部分展示
   - PCB 布局思路
   - 设计中遇到的挑战和解决方案

3. **QMK 固件**（1.5 分钟）
   - keymap.c 的键位设计逻辑
   - OLED/RGB/编码器的配置
   - 编译和刷写过程

4. **通信协议**（1 分钟）
   - I2C（OLED）的工作方式
   - USB HID 键码上报过程
   - WS2812B 数据格式

**PPT 建议**：5-6 页，每页有图示或示意图

---

### 第四部分：实物演示 (3 分钟)

**目标**：让观众亲眼看到你的作品

**演示流程 Demo Script:**

```
1. 展示外观（10 秒）
   "这是我的宏键盘成品，4x4 布局，带旋钮和 OLED 屏幕"

2. 基础按键功能（30 秒）
   打开记事本，逐一按几个键
   "按下不同的键，可以输入数字和运算符"

3. 旋钮功能（20 秒）
   旋转调节音量，按下静音
   "旋转旋钮可以调节系统音量"

4. OLED 显示（20 秒）
   按住功能键切换层
   "OLED 屏幕显示当前所在的层"

5. RGB 灯效（30 秒）
   切换多种灯效模式
   "RGB 灯效支持彩虹、呼吸、渐变等多种模式"

6. 层切换（20 秒）
   演示功能层的媒体控制功能
   "按住功能键切换到媒体层，可以控制播放"

7. VIA 改键（30 秒，可选）
   打开 VIA 软件，演示运行时改键
   "通过 VIA 可以在不重新编译的情况下修改键位"
```

**演示备份方案 Backup Plan:**
- 如果实物出故障，准备功能演示视频
- 如果电脑连接有问题，准备功能截图

---

### 第五部分：总结与反思 (2 分钟)

**目标**：展示你的工程思维和成长

- **学到了什么**：3 个最重要的技术收获
- **最有价值的技能**：硬件设计？调试能力？文档撰写？
- **不足与改进**：如果重来会怎么做得更好？
- **未来方向**：如果继续做下去，想添加什么功能？

**PPT 建议**：1-2 页

---

### 第六部分：问答环节 (5 分钟)

**常见问题准备 Common Questions to Prepare:**

| 问题类型 Question Type | 示例 Example |
|---|---|
| 原理类 | "矩阵扫描的扫描频率是多少？" |
| 设计决策 | "为什么选择 STM32 而不是 ATmega32U4？" |
| 调试经历 | "最棘手的 bug 是什么？怎么解决的？" |
| 成本控制 | "总成本多少？能不能更便宜？" |

## PPT 制作建议 | PPT Design Tips

### 设计原则

| 原则 Principle | 说明 Explanation |
|---|---|
| 少字多图 | 每页不超过 5 行文字，多用图示和照片 |
| 代码精选 | 只展示最关键的代码片段（不超过 10 行） |
| 配色统一 | 使用一致的配色方案 |
| 字体大小 | 标题 >= 28 号，正文 >= 20 号 |
| 动画适度 | 适当使用动画，但不要花哨 |

### 推荐结构

```
Slide 1:  封面（项目名称、姓名、日期）
Slide 2:  功能总览（带成品照片）
Slide 3:  技术栈概览
Slide 4:  矩阵扫描原理图
Slide 5:  PCB 设计展示
Slide 6:  QMK 固件架构
Slide 7:  通信协议（I2C/USB HID）
Slide 8:  RGB 灯效与 OLED 显示
Slide 9:  调试过程中遇到的问题
Slide 10: 成本分析
Slide 11: 学习总结与反思
Slide 12: 未来拓展方向
Slide 13: 致谢 + Q&A
```
