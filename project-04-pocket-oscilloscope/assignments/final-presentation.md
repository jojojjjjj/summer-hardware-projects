| 项目 | 说明 |
|------|------|

## 展示内容要求 | Presentation Content Requirements

### PPT 结构 (5 分钟)

| 页码 | 内容 | 建议时间 |
|------|------|---------|
| 1 | 封面：项目名称、姓名、日期 | 15 秒 |
| 2 | 项目简介：做什么、为什么做、挑战在哪 | 30 秒 |
| 3 | 系统架构：硬件框图 + 软件架构 | 45 秒 |
| 4 | 硬件设计：原理图、关键电路、PCB (如有) | 45 秒 |
| 5 | 软件设计：核心算法 (触发、测频)、代码结构 | 45 秒 |
| 6 | 成果展示：波形截图、功能演示视频、性能指标 | 45 秒 |
| 7 | 挑战与解决：遇到的困难及解决方法 | 30 秒 |
| 8 | 总结与收获：学到的知识、下一步计划 | 30 秒 |

### PPT 制作要求

- 每页不超过 5 行文字，以图片和图表为主
- 使用清晰的配色方案，字体至少 24pt
- 硬件部分必须有原理图或接线照片
- 软件部分必须有核心代码片段或流程图
- 成果部分必须有波形显示截图
- 准备一个备用方案（录屏视频），以防现场演示失败

### PPT 模板建议

```
配色方案：
  背景色：深蓝色 (#1a1a2e)
  强调色：绿色 (#00ff00) (模拟示波器荧光)
  文字色：白色 (#ffffff)
  标题色：亮蓝色 (#00aaff)
```

## 现场演示要求 | Live Demo Requirements

### 必做演示 (Must Demo)

1. **波形显示：** 用信号发生器输入正弦波，TFT 屏幕显示清晰波形
2. **频率测量：** 显示测量到的频率值，与信号发生器显示对比
3. **时基切换：** 按键切换时基，展示波形缩放效果
4. **触发模式：** 展示至少两种触发模式的区别

### 加分演示 (Bonus Demo)

5. **信号发生器：** AD9833 输出信号，示波器自收自发
6. **自动量程：** 改变信号频率，展示自动量程调整
7. **PCB 实物：** 展示自行设计的 PCB 板（如果完成）
8. **外壳实物：** 展示 3D 打印的外壳（如果完成）

### 演示注意事项

- 提前 30 分钟到场调试设备
- 准备好所有信号线和电源线
- 带一根备用 USB 线
- 信号发生器提前校准好频率
- 如果某项功能失败，不要慌张，讲讲你的排查思路

## 🌐 项目展示网站 | Project Demo Website

**这是最终评分的重要组成部分。** 你必须创建一个在线可访问的项目展示网站。

**This is a critical part of your final grade.** You must create an online-accessible project demo website.

### 要求 | Requirements

1. **部署平台** — 使用以下任一免费平台部署：
   - GitHub Pages（推荐）— 推送 HTML 到仓库的 `docs/` 分支或 `gh-pages` 分支
   - Vercel — 连接 GitHub 仓库自动部署
   - Netlify — 拖拽文件夹即可部署
   - 其他可公网访问的平台均可

2. **网站内容必须包含：**
   - [ ] 项目介绍（名称、目标、核心功能）
   - [ ] 硬件展示（BOM清单、实物照片/渲染图、接线图、电路原理图）
   - [ ] 软件架构（系统框图、模块说明、关键代码片段）
   - [ ] 演示视频或 GIF（展示示波器测量实际信号的效果）
   - [ ] 性能指标（实测带宽、采样率、波形截图）
   - [ ] 12天学习心得与总结
   - [ ] B站参考视频链接列表

3. **技术建议：**
   - 最简方案：一个 `index.html` 文件 + CSS 样式即可
   - 可使用 GitHub Pages 的 Jekyll 主题快速生成
   - 推荐工具：Markdown + 静态网站生成器（如 VitePress、Hugo）

### 提交方式 | Submission

- 在最终展示前，将网站 URL 提交给老师
- URL 格式示例：`https://你的用户名.github.io/project-name/`
- 老师将通过访问该 URL 进行评分

### 评分标准 | Scoring

| 维度 | 分值 | 说明 |
|------|------|------|
| 网站可访问性 | 5分 | URL可正常打开，无404错误 |
| 内容完整性 | 5分 | 包含所有要求的内容模块 |
| 视觉呈现 | 3分 | 页面整洁、排版合理、图文并茂 |
| 技术展示深度 | 2分 | 对核心技术的解释清晰准确 |
| **小计** | **15分** | |

## 评委提问准备 | Q&A Preparation

评委可能会问以下问题，请提前准备：

**基础问题：**
- "请简要解释你的示波器是如何工作的？"
- "ADC 的采样率是多少？是如何确定的？"
- "触发是怎么实现的？为什么需要触发？"

**进阶问题：**
- "如果要测量 1MHz 的信号，需要怎么改进？"
- "你的频率测量误差是多少？误差来源是什么？"
- "这个项目中你遇到的最大技术挑战是什么？"

**拓展问题：**
- "如果给你更多时间和预算，你会增加什么功能？"
- "这个项目对你未来的学习方向有什么影响？"
- "你会推荐其他同学做这个项目吗？为什么？"
