# 详细评分标准 | Detailed Grading Rubric

## 评分总览 | Grading Overview

| 评估维度 Dimension | 权重 Weight | 评分人 Evaluator |
|-------------------|-----------|-----------------|
| 技术实现 Technical Implementation | 35% | 导师 |
| 文档质量 Documentation | 9% | 导师 |
| 演示展示 Presentation | 17% | 导师 + 评委 |
| 进度汇报 Check-ins | 17% | 导师 |

## 1. 技术实现 (40%) | Technical Implementation

### 1.1 核心功能 (25分) | Core Features (25 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 21-25 (优秀 Excellent) | 示波器所有核心功能正常：实时波形显示、触发稳定、时基可调、频率测量准确、自动量程工作正常 |
| 16-20 (良好 Good) | 波形显示正常，基本触发功能可用，时基可调，频率测量大致准确 |
| 11-15 (及格 Pass) | 能在 TFT 上显示波形，但触发不稳定或频率测量有明显误差 |
| 6-10 (需改进 Needs Work) | 能采集 ADC 数据并输出，但波形显示不完整或无法正常工作 |
| 0-5 (未完成 Incomplete) | 未能实现基本的数据采集功能 |

### 1.2 信号调理电路 (10分) | Signal Conditioning (10 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 9-10 (优秀) | 运放电路设计合理，偏置电压稳定，输入阻抗匹配，信号不失真 |
| 7-8 (良好) | 运放电路基本工作，偶有信号失真或偏置不稳 |
| 5-6 (及格) | 有运放电路设计，但效果不理想 |
| 3-4 (需改进) | 运放电路搭建未完成或严重失真 |
| 0-2 (未完成) | 未尝试信号调理电路 |

### 1.3 拓展功能 (5分) | Extension Features (5 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 5 (优秀) | 信号发生器 (AD9833) 功能正常，可输出正弦/方波/三角波，频率可调 |
| 3-4 (良好) | 信号发生器基本功能可用，但波形质量或频率精度不足 |
| 1-2 (及格) | 尝试了信号发生器但未完全实现 |
| 0 (未完成) | 未尝试拓展功能 |

## 2. 文档质量 (10%) | Documentation

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 9-10 (优秀) | README 完整规范（含项目说明、接线图、使用方法）；代码注释清晰；有原理说明文档 |
| 7-8 (良好) | README 基本完整；代码有注释；有基本文档 |
| 5-6 (及格) | 有 README；代码有部分注释 |
| 3-4 (需改进) | README 不完整或代码缺少注释 |
| 0-2 (未完成) | 无 README 或几乎没有文档 |

---

## 3. 演示展示 (20%) | Presentation

### 3.1 现场演示 (12分) | Live Demo (12 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 10-12 (优秀) | 现场演示流畅，能稳定显示波形，操作熟练；能用信号发生器演示测量并读数 |
| 7-9 (良好) | 现场演示基本成功，波形能显示，但操作偶有卡顿 |
| 4-6 (及格) | 能演示基本功能，但需要多次尝试 |
| 0-3 (需改进) | 演示不成功或无法正常工作 |

### 3.2 技术讲解 (8分) | Technical Explanation (8 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 7-8 (优秀) | PPT 内容充实，逻辑清晰；能清楚解释示波器工作原理和设计决策；回答评委提问有理有据 |
| 5-6 (良好) | PPT 基本完整，能解释主要技术原理；回答问题基本正确 |
| 3-4 (及格) | 有 PPT 但内容较浅；回答问题需要提示 |
| 0-2 (需改进) | 无 PPT 或无法解释技术原理 |

---

## 4. 进度汇报 (20%) | Check-ins

### 4.1 第一周进度报告 (10分) | Week 1 Check-in (10 pts)

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 9-10 (优秀) | 报告内容详实，有清晰的阶段总结；配图/配代码；有深入的反思和问题分析；计划合理 |
| 7-8 (良好) | 报告完整，有总结和反思；计划基本合理 |
| 5-6 (及格) | 有报告，但内容较简略；缺乏深入反思 |
| 3-4 (需改进) | 报告不完整或过于简略 |
| 0-2 (未完成) | 未提交报告 |

### 4.2 第二周进度报告 (10分) | Week 2 Check-in (10 pts)

评分标准同上，重点关注 Phase 2 的学习成效和功能实现进度。

## 5. 团队协作 (10%) | Collaboration

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 9-10 (优秀) | Git 提交记录规范，commit message 清晰；积极帮助同学；分工合理，合作高效 |
| 7-8 (良好) | 有较规范的 Git 记录；偶尔帮助同学 |
| 5-6 (及格) | 有基本的 Git 记录；参与度一般 |
| 3-4 (需改进) | Git 记录很少或不规范；较少参与团队活动 |
| 0-2 (未完成) | 无 Git 记录或完全不参与团队协作 |

## 6. Demo 网站 (13%/15分) | Project Demo Website

详见 `assignments/rubric.md` 第 6 节。要求学生创建项目展示网站，包含以下内容：

| 分数 Score | 标准 Criteria |
|-----------|-------------|
| 13-15 (优秀) | 网站可访问且加载完整；包含全部 7 项内容（项目介绍、硬件展示、软件架构、演示视频/GIF、性能指标、学习心得、B站参考链接）；页面美观，技术解释清晰 |
| 10-12 (良好) | 网站可访问；包含 5-6 项内容；页面基本整洁 |
| 7-9 (及格) | 网站可访问；包含 3-4 项内容 |
| 4-6 (需改进) | 网站偶尔可访问或内容不足 3 项 |
| 0-3 (未完成) | 无法访问或未提交网站 |

完整评分细则见 `assignments/rubric.md` 中的第 6 节（6.1 网站可访问性 / 6.2 内容完整性 / 6.3 视觉呈现与技术深度）。
