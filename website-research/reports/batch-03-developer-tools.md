## 1. Tailwindcss.com — 功能即展示的教科书

### 🎯 设计哲学
Tailwind CSS的官网本身就用Tailwind构建，是最好的dogfooding案例。每个特性都通过**实时交互demo**展示，而非文字描述。

### 🎨 视觉风格
| 维度 | 分析 |
|------|------|
| **配色** | 深色背景 + 天蓝色(#38bdf8)强调色 + oklch P3广色域色彩 |
| **字体** | Inter Variable（正文）+ IBM Plex Mono（代码）+ Ubuntu Mono Bold（装饰） |
| **间距** | 紧凑型，信息密度高 |
| **图片** | 真实网站截图showcase + 内嵌交互demo |

### ✨ 交互与动效
- **实时CSS demo**：每个Tailwind特性都配有可交互的实时demo
- **响应式预览**：展示Mobile/sm/md/lg/xl断点的实时切换
- **Dark/Light切换**：一键切换明暗主题
- **3D Transform demo**：交互式3D变换展示
- **代码实时渲染**：左侧代码 → 右侧渲染效果的实时对应
- **渐变编辑器**：拖拽式渐变创建器
- **Showcase墙**：展示OpenAI、Reddit、Shopify、NASA等用Tailwind构建的网站

### 🔧 技术实现
```
技术栈：
- Next.js（自研框架dogfooding）
- Tailwind CSS v4.3（自用）
- Inter Variable / IBM Plex Mono 字体
- 内嵌交互demo（React组件）
- Algolia搜索（⌘K快捷键）
- 完善的RSS/Atom/JSON Feed
```

### 📐 可借鉴的设计模式
1. **特性=交互demo** — 每个项目特性用可交互demo展示，而非纯文字
2. **代码↔渲染对照** — 左代码右效果的双栏布局
3. **Showcase墙** — 展示使用本项目技术构建的作品
4. **⌘K搜索** — 快捷键搜索导航

### ⭐ 推荐指数：★★★★★
**最适合**：技术特性展示方式、Showcase页面设计

## 2. Supabase.com — 开源社区驱动的BaaS

### 🎯 设计哲学
Supabase的网站完美平衡了技术深度和视觉吸引力。"Build in a weekend, Scale to millions"是其核心价值主张。

### 🎨 视觉风格
| 维度 | 分析 |
|------|------|
| **配色** | 深色背景(#1E1E1E) + 绿色品牌色 + SVG装饰线 |
| **字体** | 现代无衬线 |
| **间距** | 标准SaaS间距，产品section清晰分隔 |
| **图片** | 产品截图（暗色/亮色双版本）+ SVG装饰图 + 客户logo |

### ✨ 交互与动效
- **Logo滚动墙**：Mozilla、GitHub、1Password等知名客户logo无限滚动
- **产品模块展示**：Database/Auth/Functions/Storage/Realtime/Vector 每个独立卡片
- **亮/暗双版本**：产品截图同时提供light和dark版本
- **Twitter评价墙**：真实Twitter用户评价无限滚动
- **客户案例轮播**：Quivr/Mobbin/Mozilla等案例卡片
- **GitHub Star计数**：实时展示GitHub Star数（103K）
- **Template展示**：社区构建的模板展示

### 🔧 技术实现
```
技术栈：
- Next.js
- 多种Favicon尺寸（16x16到196x196）
- Apple Touch Icon全套
- RSS Feed
- homepage.md（直接可读的首页内容）
```

### 📐 可借鉴的设计模式
1. **Logo信任墙** — 知名客户/合作伙伴logo无限滚动
2. **产品模块拆解** — 每个功能模块独立section + 图标 + 简短描述
3. **社区评价墙** — 真实社交媒体评价
4. **开源Star展示** — 展示GitHub Star数和排名

### ⭐ 推荐指数：★★★★☆
**最适合**：合作伙伴/学校logo墙、功能模块展示

## 3. shadcn/ui — 组件即设计的哲学

### 🎯 设计哲学
shadcn/ui的首页是一个"活的设计系统"——直接在landing page中展示所有组件的实际效果，无需截图。

### 🎨 视觉风格
| 维度 | 分析 |
|------|------|
| **配色** | 白色/浅色背景 + 深色文字 + 微妙的边框和阴影 |
| **字体** | Geist Sans（Vercel自研） |
| **间距** | 标准dashboard间距 |
| **图片** | Dashboard截图（亮色/暗色双版本）+ 实际可交互组件 |

### ✨ 交互与动效
- **全功能Dashboard预览**：直接在页面中嵌入一个完整的dashboard UI
- **表单组件实时交互**：支付表单、设置面板、聊天界面全部可交互
- **亮/暗主题切换**：一键切换整个预览的主题
- **113K计数器**：展示GitHub Star数
- **组件分类导航**：清晰的产品分类菜单

### 📐 可借鉴的设计模式
1. **活组件预览** — 直接在页面中嵌入可交互的组件/项目demo
2. **Dashboard风格展示** — 用dashboard布局展示项目数据
3. **明暗双主题** — 同时展示light/dark两种模式

### ⭐ 推荐指数：★★★★☆
**最适合**：项目详情页的交互预览

## 4. buildspace.so — 极简退役页面

### 🎯 设计哲学
buildspace已退役，但其极简的退役页面本身就是一种设计宣言。"hi. this was buildspace." 重复四次，传达一种怀旧和仪式感。

### 🎨 视觉风格
| 维度 | 分析 |
|------|------|
| **配色** | 黑色背景 + 白色文字 |
| **字体** | 大号粗体 |
| **间距** | 极简，几乎只有文字 |
| **图片** | 无 |

### 📐 可借鉴的设计模式
1. **极简宣言** — 用最少的元素传达最核心的信息
2. **重复即强调** — 标题重复4次，创造视觉节奏

### ⭐ 推荐指数：★★☆☆☆（已退役，参考价值有限）

## 5. Flipperzero.one — 硬件规格的精准呈现

### 🎯 设计哲学
Flipper Zero的网站专注于硬件规格展示——每个传感器、接口、参数都清晰列出，用极简的排版让技术细节变得易读。

### 🎨 视觉风格
| 维度 | 分析 |
|------|------|
| **配色** | 橙色品牌色 + 深色背景（产品页）+ 白色/浅灰背景（规格页） |
| **字体** | 清晰无衬线 |
| **间距** | 规格表用精确的排版，参数对齐 |
| **图片** | 产品渲染图 + 功能示意图 |

### ✨ 交互与动效
- **硬件规格详表**：MCU型号/频率/Flash/SRAM精确展示
- **功能图标分组**：Sub-1GHz/Bluetooth/Infrared/NFC/RFID/GPIO/USB等分类展示
- **支持协议列表**：每个接口下清晰列出支持的协议
- **精确参数**：频率范围/功率/灵敏度/数据速率等精确数值

### 📐 可借鉴的设计模式
1. **硬件规格表** — MCU型号/频率/Flash/接口的精确展示
2. **功能图标分组** — 每个传感器/功能用独立图标+标题+描述
3. **协议/技术列表** — 清晰列出支持的技术和标准

### ⭐ 推荐指数：★★★★★
**最适合**：项目硬件参数展示、技术栈清单展示
