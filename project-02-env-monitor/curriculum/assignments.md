# 作业与评分标准 | Assignments & Grading

> 本项目作业设计与评分标准 | Project Assignments and Grading Criteria

---

## 📋 作业概览 | Assignments Overview

### 每日作业 | Daily Assignments

每天课程结束时提交,包括:

1. **基础作业** (必做)
   - 代码练习
   - 实验报告
   - 概念问题

2. **进阶作业** (选做)
   - 拓展研究
   - 创新实现
   - 深度思考

### 阶段性作业 | Phase Assignments

**Week 1 Check-in** (Day 6)
- 本地功能演示
- 代码提交
- 进度报告

**Week 2 Check-in** (Day 9)
- 远程监控演示
- 系统集成报告
- 中期评估

**Final Presentation** (Day 12)
- 项目演示
- 技术文档
- 最终答辩

## 📊 评分标准 | Grading Rubric

### 总体评分分布 | Overall Grading Distribution

| 类别 | 权重 | 说明 |
|------|------|------|
| **技术实现** | 40% | 功能完整度、代码质量、系统稳定性 |
| **文档质量** | 20% | README、注释、图表、技术文档 |
| **演示展示** | 20% | Final Demo、演讲技巧、问答表现 |
| **进度汇报** | 10% | Weekly Check-in质量、进度跟踪 |
| **团队协作** | 10% | Git使用、代码规范、协作精神 |

### 详细评分标准 | Detailed Rubric

#### 1. 技术实现 (40分)

**功能完整性 (15分)**
- 优秀 (13-15): 所有计划功能实现,运行稳定
- 良好 (10-12): 核心功能实现,有少量问题
- 及格 (7-9): 基本功能实现,存在明显问题
- 不及格 (0-6): 功能不完整或无法运行

**代码质量 (15分)**
- 优秀 (13-15): 代码规范、结构清晰、有完整注释
- 良好 (10-12): 代码较规范、有基本注释
- 及格 (7-9): 代码可用但不够规范
- 不及格 (0-6): 代码混乱、难以维护

**系统稳定性 (10分)**
- 优秀 (9-10): 长时间运行无崩溃,异常处理完善
- 良好 (7-8): 偶有小问题,不影响使用
- 及格 (5-6): 有明显bug,但基本可用
- 不及格 (0-4): 频繁崩溃或无法使用

#### 2. 文档质量 (20分)

**README (5分)**
- 项目概述清晰
- 安装步骤完整
- 使用说明详细

**技术文档 (10分)**
- 架构设计图
- API文档
- 故障排查指南

**代码注释 (5分)**
- 关键函数有注释
- 复杂逻辑有说明
- 变量命名清晰

#### 3. 演示展示 (20分)

**演示效果 (10分)**
- 流畅无卡顿
- 功能展示全面
- 时间控制得当

**演讲质量 (5分)**
- 表达清晰
- 逻辑性强
- 回答准确

**视觉材料 (5分)**
- 幻灯片/图表
- 演示视频(如有)
- 产品照片

#### 4. 进度汇报 (10分)

**Week 1 Check-in (5分)**
- 按时提交
- 内容真实
- 问题分析

**Week 2 Check-in (5分)**
- 系统集成状态
- 遇到的挑战
- 解决方案

#### 5. 团队协作 (10分)

**Git使用 (5分)**
- 提交规范
- 分支管理
- 代码审查

**协作精神 (5分)**
- 互相帮助
- 知识分享
- 积极参与

## 📝 作业提交要求 | Submission Requirements

### 代码提交 | Code Submission

```bash
# Git提交规范
git add .
git commit -m "feat: 完成BME680传感器读取"

# 提交类型
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式
refactor: 重构
test: 测试
chore: 构建/工具
```

### 报告格式 | Report Format

**实验报告模板:**

```markdown
# 实验报告: [实验名称]

## 一、实验目的
- 目标1
- 目标2

## 二、实验环境
- 硬件: ESP32, BME680
- 软件: MicroPython, Thonny

## 三、实验步骤
1. 步骤描述
2. 代码片段
3. 预期结果

## 四、实验结果
- 数据记录
- 截图展示
- 结果分析

## 五、遇到的问题
- 问题描述
- 解决方案
- 经验总结

## 六、思考与总结
- 心得体会
- 改进建议
```
